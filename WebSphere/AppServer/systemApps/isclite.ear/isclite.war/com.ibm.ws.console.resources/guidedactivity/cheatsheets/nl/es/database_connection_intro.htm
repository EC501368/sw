Suposiciones

Se supone que va a instalar una aplicación que necesita acceder de forma segura a los datos de una base de datos relacional. Para obtener más información sobre esta tarea, consulte las siguientes fuentes en el Information Center:
  • Configuración de proveedores de JDBC y de orígenes de datos
  • Despliegue de aplicaciones de acceso a datos
  • Información sobre los recursos de acceso a datos

Antes de empezar

  • Compruebe que la base de datos y el proveedor de JDBC están soportados por WebSphere Application Server. Para obtener más información consulte Valores mínimos necesarios de orígenes de datos específicos del proveedor en el Information Center.
  • Cree y configure las bases de datos y tablas necesarias y compruebe que puede conectarse a las bases de datos utilizando las herramientas del proveedor de la base de datos.
  • Instale los controladores del cliente de bases de datos en el mismo nodo que el WebSphere Application Server que accede a la base de datos.

Información que necesita saber

  • Para configurar el proveedor de JDBC, debe conocer la vía de acceso del sistema de archivos a los controladores del cliente de bases de datos y el nombre de clase de la implementación [ConnectionPoolDataSource o XADataSource] para el proveedor.
  • Conozca los siguientes requisitos de los recursos de la aplicación:
    • Orígenes de datos a los que hace referencia la aplicación.
    • Si la aplicación contiene o no enterprise beans u otros componentes Web para acceder a la base de datos.
    • Si se necesitan credenciales de seguridad para conectar a las bases de datos.
