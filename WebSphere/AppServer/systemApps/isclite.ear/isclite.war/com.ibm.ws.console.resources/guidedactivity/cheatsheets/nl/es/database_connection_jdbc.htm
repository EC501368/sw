Consejos y sugerencias

Existe una sección Ámbito en la página de colecciones de proveedores de JDBC. Si utiliza la edición WebSphere Application Server Network Deployment, verá que aparece un conjunto más avanzado de opciones en el panel de selección de Ámbito. Si selecciona el ámbito de nodo (es decir, Ámbito: Cell="MiCélula",Node="MiNodo") significa que los proveedores de JDBC que configure estarán disponibles para todos los servidores de aplicaciones que se ejecuten en el nodo especificado. Se recomienda utilizar el ámbito de nodo con las ediciones de WebSphere Application Server-Express y WebSphere Application Server. Puede contraer la sección de ámbito para ocultar los campos de selección de ámbito si no tiene previsto realizar configuraciones más avanzadas de ámbitos de recursos. Para obtener más información sobre la configuración avanzada de ámbitos, consulte el artículo del Information Center titulado Valores de ámbito de la consola administrativa.

Si no se suministra una plantilla con WebSphere Application Server para su base de datos compatible con JDBC, puede crear un proveedor definido por el usuario durante el paso en que se selecciona el tipo de proveedor. En el caso de proveedores de JDBC definidos por el usuario, es necesario especificar el nombre correcto de la clase de implementación en el panel Detalles. Consulte la documentación del controlador cliente del proveedor de bases de datos para determinar los nombres de la clase de implementación adecuada y del archivador Java (JAR) que se deben utilizar. Los valores por omisión de la clase de implementación y de las vías de acceso de clases para los proveedores definidos por el usuario en la consola administrativa no funcionan sin modificaciones y sólo se suministran como sugerencias.
