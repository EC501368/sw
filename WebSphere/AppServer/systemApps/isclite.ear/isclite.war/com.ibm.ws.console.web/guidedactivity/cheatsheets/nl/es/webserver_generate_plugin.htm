Consejos y sugerencias

Generar un archivo de configuración del plug-in puede llevar su tiempo

Volver a generar el archivo de configuración del plug-in para un servidor Web puede tardar en completarse. Después de regenerar el archivo de configuración del plug-in, todos los objetos de la célula administrativa utilizan sus valores más recientes, a los que puede acceder el servidor Web.

El proceso de regeneración del plug-in, tanto si se activa manualmente como si se produce de forma automática, requiere entre 30 y 60 segundos para completarse cuando el servidor de aplicaciones está en la misma máquina física o nodo que el servidor Web. En otros casos, necesita más tiempo.

El retardo es importante porque determina la rapidez con que la nueva configuración del plug-in entra en vigor. Por ejemplo, si añade una nueva vía de acceso de servicio para un servlet y regenera la configuración del plug-in, el proceso de regeneración necesita 40 segundos. Después del proceso de regeneración, un usuario podrá acceder al servlet mediante la nueva vía de acceso de servicio.
