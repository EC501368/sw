Consejos y sugerencias

  • En el caso de los cortafuegos, el cortafuegos del puerto administrativo debe tener abierto el mismo puerto para permitir que pase el archivo del plug-in.
  • La regeneración del plug-in requiere 60 segundos para completarse cuando el servidor de aplicaciones está en la misma máquina física o nodo que el servidor Web. En este caso, la duración del proceso de regeneración es la misma, tanto si el proceso se activa manualmente como si se produce automáticamente. En otros casos, el proceso necesita más tiempo.
  • Por omisión, las versiones del archivo plugin-cfg.xml que están en funcionamiento o activas residen en el directorio WAS_HOME/AppServer/config/cells/plugin-cfg.xml.
  • Regenere la configuración del plug-in después, por ejemplo, de instalar o eliminar una aplicación de empresa, de añadir o eliminar servlets y correlaciones de una aplicación concreta, o de cambiar la configuración del plug-in, de un host virtual o de un transporte. Si no regenera el plug-in después de incorporar una nueva aplicación podría aparecer un error 404 Archivo no encontrado cuando intente acceder a la nueva aplicación Web.
  • La regeneración de la configuración del plug-in puede escribir encima de cambios de configuración manuales que desea conservar. Antes de realizar esta tarea, conozca las consecuencias que se describen en el artículo del Information Center titulado Configuración de un plug-in del servidor Web para comunicarse con un servidor de aplicaciones que se ejecute en una plataforma distribuida.
