Gestor de carga de trabajo dinámica (DWLM)

El gestor de carga de trabajo dinámica (DWLM o Dynamic workload manager) aplica los mismos principios que el que el gestor de carga de trabajo (WLM). Tanto el DWLM como el WLM realizan el direccionamiento en base a un sistema de pesos, que establece un sistema de direccionamiento por prioridades. En lugar de establecer de forma manual un peso estático como en WLM, DWLM otorga al sistema la posibilidad de modificar los pesos de forma que estén actualizados de acuerdo a los objetivos empresariales.

Para ver esta página de la consola administrativa, pulse Servidores > Clústeres > nombre_clúster > Gestión de carga de trabajo dinámica (DWLM) o Servidores > Clústeres dinámicos > nombre_clúster > Gestión de carga de trabajo dinámica (DWLM).

Seleccionando el recuadro de selección otorga al sistema la posibilidad de modificar los pesos WLM, si es necesario. Si no se selecciona el recuadro de selección, inhabilita la funcionalidad DWLM.

Tenga en cuenta los siguientes aspectos al decidir si desea habilitar el DWLM:
  • ¿Necesita el clúster un peso estático o desea que el peso del clúster fluctúe con la capacidad del tráfico?

    Si está utilizando operaciones dinámicas para él clúster, debe establecer objetivos empresariales y utilizar políticas de uso para definir un umbral. En este caso, establecer un peso estático no es útil. Si está trabajando en un entorno estático, establezca un peso estático.

  • ¿Tiene planificado utilizar operaciones dinámicas?

    Si tiene la intención de utilizar la modalidad automática para las operaciones dinámicas, establecer pesos WLM estáticos en cualquiera de los clústeres dinámicos podría impedir que las operaciones dinámicas funcionasen correctamente.

  • ¿Desea que en un clúster se restrinja la realización de un cierto trabajo?

    Si se tiene un clúster que no se desea dedicarlo para la carga de trabajos actual, se puede crearlo como un clúster estático y definirle un peso WLM estático. Estableciendo un peso estático, está estableciendo un porcentaje a parte del clúster para otro trabajo.

  • ¿Está interesado en utilizar el equilibrado de sesiones HTTP?

    Con el equilibrado de sesiones HTTP, el producto puede equilibrar de forma activa y dinámica la distribución de sesiones HTTP entre los servidores de aplicaciones.
