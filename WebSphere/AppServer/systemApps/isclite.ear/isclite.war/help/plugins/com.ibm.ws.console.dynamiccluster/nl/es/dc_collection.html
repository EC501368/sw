Colección de clústeres dinámicos

Utilice esta página para ver clústeres dinámicos que ya existen. Los miembros de los clústeres dinámicos, o instancias de servidor, residen en un nodo dentro de un grupo de nodos. Los clústeres dinámicos, así como sus grupos de nodos asociados, se utilizan para realizar la ubicación de aplicaciones cuando el entorno se está ejecutando en una modalidad automática.

Para ver esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres de dinámicos.

Es necesario poseer privilegios de supervisor para ver la información de los clústeres dinámicos. Consulte el centro de información para obtener más información sobre otros roles administrativos y sobre privilegios.

Nombre

Especifica un nombre para el clúster dinámico. El nombre de un clúster dinámico debe ser exclusivo entre todos los clústeres en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre no puede empezar con un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Tipo

Especifica los tipos de servidores que se están utilizando en el clúster dinámico.

La siguiente lista contiene los tipos de servidor:

WebSphere® Application Server
Especifica que el clúster dinámico contiene servidores de aplicaciones que ejecutan WebSphere Application ServerIntelligent Management.
Servidor de perfiles Liberty
Especifica que el clúster dinámico contiene servidores de perfiles Liberty.
Servidor Apache
Especifica que el clúster dinámico contiene servidores Apache.
Servidor HTTP personalizado
Especifica que el clúster dinámico contiene servidores HTTP.
Servidor JBoss
Especifica que el clúster dinámico contiene servidores en que se ejecuta el servidor de aplicaciones de JBoss.
Direccionador On Demand
Especifica que se está creando un clúster dinámico de direccionadores On Demand.
Servidor PHP
Especifica que el clúster dinámico contiene servidores en que se ejecuta PHP.
Servidor Apache Tomcat
Especifica que el clúster dinámico contiene servidores en que se ejecuta Apache Tomcat.
Servidor WebSphere Application Server Community Edition
Especifica que el clúster dinámico contiene servidores en que se ejecuta WebSphere Application Server Community Edition.
Servidor BEA WebLogic
Especifica que el clúster dinámico contiene servidores en que se ejecuta BEA WebLogic.
Servidores de aplicaciones de WebSphere externos
Especifica que el clúster dinámico contiene servidores que están en otras células. Estas células podrían estar ejecutando otras versiones de WebSphere Application Server, como las versiones 5.1 ó 6.0. Estos clústeres estáticos se pueden correlacionar con el clúster dinámico.

Modalidad operativa

Especifica el nivel en que interactúa el clúster dinámico con el entorno de operaciones dinámicas.

  • Manual: en la modalidad manual, el clúster dinámico no es distinto de los entornos de servidores de aplicaciones normales con clústeres estáticos. No hay interacción con el entorno de operaciones dinámicas.
  • Supervisada: en la modalidad supervisada, el entorno de Intelligent Management proporciona información y recomienda acciones que el administrador puede utilizar.
  • Automática: en la modalidad automática, el entorno de Intelligent Management toma decisiones de forma automática. En la modalidad automática, el entorno de operaciones dinámicas maneja las ubicaciones y la preparación.

Seleccione los clústeres de los que desee cambiar su modalidad operacional, seleccione el tipo de modalidad que desea definir y pulse Establecer modalidad.
