Valores del clúster dinámico

Utilice esta página para ver configuraciones de clústeres dinámicos existentes.

Para ver esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres dinámicos > nombre_clúster_dinámico.

Es necesario poseer privilegios de supervisor para ver la configuración de los clústeres dinámicos existentes.

Nombre

Especifica un nombre para el clúster dinámico. El nombre de un clúster dinámico debe ser exclusivo entre todos los clústeres en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre no puede empezar con un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Tipo

Especifica los tipos de servidores que se están utilizando en el clúster dinámico.

La siguiente lista contiene los tipos de servidor:

WebSphere® Application Server
Especifica que el clúster dinámico contiene servidores de aplicaciones que ejecutan WebSphere Application ServerIntelligent Management.
Servidor de perfiles Liberty
Especifica que el clúster dinámico contiene servidores de perfiles Liberty.
Servidor Apache
Especifica que el clúster dinámico contiene servidores Apache.
Servidor HTTP personalizado
Especifica que el clúster dinámico contiene servidores HTTP.
Servidor JBoss
Especifica que el clúster dinámico contiene servidores en que se ejecuta el servidor de aplicaciones de JBoss.
Direccionador On Demand
Especifica que se está creando un clúster dinámico de direccionadores On Demand.
Servidor PHP
Especifica que el clúster dinámico contiene servidores en que se ejecuta PHP.
Servidor Apache Tomcat
Especifica que el clúster dinámico contiene servidores en que se ejecuta Apache Tomcat.
Servidor WebSphere Application Server Community Edition
Especifica que el clúster dinámico contiene servidores en que se ejecuta WebSphere Application Server Community Edition.
Servidor BEA WebLogic
Especifica que el clúster dinámico contiene servidores en que se ejecuta BEA WebLogic.
Servidores de aplicaciones de WebSphere externos
Especifica que el clúster dinámico contiene servidores que están en otras células. Estas células podrían estar ejecutando otras versiones de WebSphere Application Server, como las versiones 5.1 ó 6.0. Estos clústeres estáticos se pueden correlacionar con el clúster dinámico.

Número de instancias en ejecución

Especifica un valor que se genera de forma automática en base al número de instancias de servidor que hay en ejecución en el clúster dinámico.

Por ejemplo, un clúster dinámico puede tener tres aplicaciones en ejecución, con dos instancias en ejecución en cada uno de los cuatro miembros del clúster dinámico. El número de instancias es de 8.

Modalidad operativa

Especifica el nivel en que interactúa el clúster dinámico con el entorno de operaciones dinámicas:

  • Manual: en la modalidad manual, el clúster dinámico no es distinto de los entornos de servidores de aplicaciones normales con clústeres estáticos. No hay interacción con el entorno de operaciones dinámicas.
  • Supervisada: en la modalidad supervisada, el entorno de Intelligent Management proporciona información y recomienda acciones que el administrador puede utilizar.
  • Automática: en la modalidad automática, el entorno de Intelligent Management toma decisiones de forma automática. En la modalidad automática, el entorno de operaciones dinámicas maneja las ubicaciones y la preparación.

Puede cambiar la modalidad operativa en la consola administrativa pulsando Servidor > Clústeres dinámicos.

Habilitar sustitución por anomalía de recuperación de anotaciones cronológicas de transacciones

Especifica que para el componente del servicio de transacciones, se habilita o inhabilita a efectos de recuperación la sustitución por anomalía de anotaciones cronológicas de transacciones. Este campo únicamente es válido para clústeres dinámicos de servidores de aplicaciones de WebSphere.

Por omisión está inhabilitada. Si habilita este valor, las propiedades del servicio de transacciones que se necesitan para la recuperación de igual de aplicaciones con anomalías deben estar configuradas correctamente. Si estas propiedades están correctamente configuradas, se realizará la recuperación por anomalía de las anotaciones cronológicas de las transacciones si se produce una anomalía en el servidor que procesa las anotaciones cronológicas de las transacciones. Si las propiedades de los servicios de transacciones necesarias para la recuperación de igual de los servidores de aplicaciones con anomalías en un clúster no se configuran correctamente, este valor se ignora.

Política de pertenencia

Especifica una expresión que se utiliza para definir los nodos que se utilizarán para el clúster dinámico. Esta opción sólo aparece si se ha elegido definir de forma automática los miembros del clúster mediante normas. Esta opción no aparece si se elige definir de forma manual los miembros del clúster.

Utilice el generador de subexpresiones para crear una Política de pertenencia. La política de pertenencia especifica los nodos que son miembros del clúster dinámico. Pulse Previsualizar pertenencia para ver qué nodos están en el clúster dinámico con la política de pertenencia definida.

Políticas de pertenencia de ejemplo

  • Para definir la pertenencia de todos los nodos en un grupo de nodos específicos: 
    node_nodegroup = 'mi_grupo_nodos'
  • Para definir la pertenencia de todos los nodos en los que se ejecuta Apache.
    node_property$APACHE_2_0 IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta Apache y PHP:
    node_property$APACHE IS NOT NULL and node_property$PHP IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta WebSphere Application Server Community Edition Versión 2.0 o posterior:
    node_property$WASCE_2.0* IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta WebSphere Application Server Community Edition Versión 2.0 únicamente:
    node_property$WASCE_2.0 IS NOT NULL
  • Para definir la pertenencia de todos los nodos que tienen un nombre que es como mi_nodo, incluidos mi_nodo_1, mi_nodo_2, etc.:
    node_name like 'mi_nodo%'
  • Para definir la pertenencia de una lista específica de nombres de host:
    node_hostname in ('nodo1.midominio.com','nodo2.midominio.com')
Las subexpresiones se pueden agrupar con paréntesis y operadores lógicos and, or o not. En el siguiente ejemplo se define la pertenencia a un nodo como los nodos que no están en el grupo de nodos mi_grupo_nodos y tienen un nombre que es como mi_nodo, incluidos mi_nodo_1, mi_nodo_2, etc.
not (node_nodegroup='mi_grupo_nodos') and node_name like 'mi_nodo%'

Número mínimo de instancias de clúster

Especifica el número mínimo de instancias en ejecución para el clúster dinámico. Si un valor mínimo es excesivo, es posible que se produzca una degradación del rendimiento.

  • Pulse Detener todas las instancias durante periodos de inactividad para habilitar el inicio lento.
    Restricción: si se está direccionando tráfico SIP (Session Initiation Protocol) al clúster dinámico, no utilice la opción Si otros clústeres dinámicos necesitan recursos, detener todas las instancias de este clúster en periodos de inactividad. Utilice una de las opciones que mantienen al menos activo un miembro del clúster todo el tiempo.
  • Si desea que siempre exista una instancia de clúster iniciada, pulse Mantener iniciada una instancia todo el tiempo. Esta es la opción predeterminada.
  • Si desea que siempre exista más de una instancia de clúster iniciada, pulse Mantener iniciadas varias instancias todo el tiempo. Si se selecciona esta opción, indique el Número de instancias que desea iniciadas todo el tiempo.

Número máximo de instancias de clúster

Especifica el número máximo de instancias de clúster que se pueden iniciar.

  • Si desea limitar el número de instancias de clúster, pulse Limitar el número de instancias que pueden iniciarse y especifique el Número de instancias máximo.
  • Si no desea limitar el número de instancias de clúster, pulse No limitar el número de instancias que pueden iniciarse.

Apilamiento vertical de instancias en el nodo

Especifica si se puede iniciar más de una instancia en un único nodo. Al configurar el apilamiento vertical, el controlador de ubicación de aplicaciones puede iniciar más de una instancia del clúster dinámico en el nodo cuando exista una carga de trabajo grande.

El apilamiento vertical sirve para mejorar situaciones de cuello de botella entre aplicaciones. Seleccione Permitir iniciar más de una instancia en el mismo nodo y escriba el valor de Número de instancias. El número de instancias es el número de servidores de aplicaciones para un clúster que se necesitan para utilizar todas las capacidades de un nodo. Además, el número que elija para el apilamiento vertical es el número de servidores que se crean automáticamente.

Si se utiliza el apilamiento vertical con nodos de destino que son de una versión del producto más antigua que la versión del gestor de despliegue, se inhabilita la adición de nuevos miembros en el mismo nodo. El apilamiento vertical únicamente está disponible en servidores de aplicaciones, servidores de WebSphere Application Server Community Edition y servidores PHP que tengan miembros de clúster que las normas definan automáticamente.

Preferencias de aislamiento

Especifica si se desea que el clúster dinámico esté en ejecución en los mismos nodos que otras instancias de clústeres dinámicos, o si desea que el clúster dinámico sea el único clúster dinámico en ejecución en el nodo. Esta opción únicamente está disponible si se elige definir de forma automática los miembros del clúster mediante normas. Si se definen los miembros del clúster de forma manual, no se tendrá la opción de habilitar el aislamiento de clúster dinámico.

  • Seleccione No hay requisitos de aislamiento si las instancias de clúster en este clúster dinámico se pueden ejecutar en el mismo nodo que otras instancias de clústeres dinámicos distintos. Esta es la opción predeterminada.
  • Seleccione Aislamiento estricto si instancias de clúster en este clúster dinámico sólo se pueden ejecutar con otras instancias del mismo clúster dinámico en el mismo nodo.
  • Seleccione Asociar con un grupo de aislamiento si la instancia de clúster se puede ejecutar con cualquier otra instancia de un clúster dinámico que esté en el mismo grupo compartido en el mismo nodo. Especifique un valor de Nombre de grupo de aislamiento. Escriba un nombre exclusivo para crear un nuevo grupo de aislamiento para este clúster dinámico. Pulse Examinar para seleccionar de una lista de grupos de aislamiento existentes. Se muestra una topología de los grupos de aislamiento actuales. Pulse sobre un nombre de un grupo de aislamiento para seleccionarlo.
