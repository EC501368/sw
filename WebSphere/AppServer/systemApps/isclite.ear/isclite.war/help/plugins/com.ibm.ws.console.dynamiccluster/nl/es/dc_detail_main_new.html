Creación de un clúster dinámico

Utilice esta página para crear clústeres dinámicos.

Pulse Servidores > Clústeres dinámicos > Nuevo para ver esta página de la consola administrativa.

Es necesario poseer privilegios de configurador para crear clústeres dinámicos. Consulte la sección de referencia en el centro de información para obtener información sobre otros roles administrativos y privilegios.

Paso 1: Entrar información del clúster dinámico
Nombre de clúster dinámico
Especifica un nombre para el clúster dinámico que sea único en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre definido debe ser exclusivo entre clústeres dinámicos y no puede comenzar por un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Correlacionar con un grupo de nodos
Especifica un grupo de nodos para vincular al clúster dinámico.

Este grupo de nodos debe tener al menos un nodo de miembro que esté en la misma versión de producto que el gestor de despliegue.

Selecciónelo de cualquiera de los grupos de nodos disponibles. Si no especifica uno, se selecciona un grupo de nodos por omisión para el clúster dinámico. Un grupo de nodos y un clúster dinámico tienen una relación estrecha. El grupo de nodos es una afiliación de nodos definida por el usuario que comparte una propiedad común. Un clúster dinámico es una clúster de servidores dentro de un grupo de nodos, cuyos miembros, o instancias de servidor, residen en un nodo de un grupo de nodos y son los objetivos de despliegue de las aplicaciones.

Preferencia local habilitada
Para habilitar o inhabilitar la optimización del direccionamiento con ámbito de nodo, seleccione el recuadro de selección Preferencia local habilitada. El valor por omisión es habilitado, que indica que, si es posible, las peticiones de EJB (Enterprise JavaBeans) se direccionan al nodo cliente. Si habilita esta función, el rendimiento mejora porque las peticiones del cliente se envían a enterprise beans locales.
Crear un dominio de duplicación para este clúster
Especifica que se crea un dominio de duplicación para el clúster dinámico. El nombre del domino de duplicación se establece en el nombre del clúster dinámico que utiliza este dominio.
Paso 2: Seleccionar una plantilla del clúster dinámico

Seleccione una plantilla de servidor de aplicaciones para utilizarla como modelo para los servidores para el nuevo clúster dinámico. Se puede seleccionar una plantilla por omisión, o correlacionar un servidor de aplicaciones que ya exista.

  • Seleccione Plantilla de servidor de aplicaciones por omisión para utilizarla como modelo para los servidores para el clúster dinámico a partir de una plantilla de servidor definida. Seleccione una plantilla de servidor de la lista.
  • Seleccione Servidor de aplicaciones existente para utilizarlo como modelo para los servidores para el clúster dinámico a partir de un servidor de aplicaciones existente. Seleccione un servidor de aplicaciones de la lista.

Las plantillas que se muestran corresponden a la plataforma y versión de producto del gestor de despliegue que esté utilizando.

Paso 3: Especificar propiedades específicas del clúster dinámico

Especifique estas propiedades para definir el comportamiento del clúster dinámico.

Número mínimo de instancias de clúster

Especifica el número mínimo de instancias en ejecución para el clúster dinámico. Si un valor mínimo es excesivo, es posible que se produzca una degradación del rendimiento.

  • Seleccione Detener todas las instancias iniciadas durante periodos de inactividad para detener las instancias de clúster cuando haya poca actividad en el clúster dinámico Escriba el valor de Tiempo de espera antes de detener instancias en minutos. Esta es la opción por omisión.
  • Seleccione Mantener iniciada una instancia todo el tiempo para mantener una instancia del clúster iniciada todo el tiempo.
  • Seleccione Mantener iniciadas varias instancias todo el tiempo para tener un determinado número de instancias del clúster iniciadas todo el tiempo. Escriba el valor de Número de instancias que desee tener iniciadas.
Número máximo de instancias de clúster
Especifica el número máximo de instancias de clúster que se pueden iniciar.
  • Seleccione Limitar número de instancias que pueden iniciarse para limitar el número de instancias de clúster. Escriba el máximo de Número de instancias.
  • Seleccione No limitar el número de instancias que pueden iniciarse si no desea limitar el número de instancias de clúster que se pueden iniciar.
Apilamiento vertical de instancias en el nodo
Especifica el número máximo de instancias de clúster que se pueden iniciar.

El apilamiento vertical sirve para mejorar situaciones de cuello de botella entre aplicaciones. Seleccione Permitir iniciar más de una instancia en el mismo nodo y escriba el valor de Número de instancias. El número de instancias es el número de servidores de aplicaciones para un clúster que se necesitan para utilizar todas las capacidades de un nodo. Además, el número que elija para el apilamiento vertical es el número de servidores que se crean automáticamente.

Si se utiliza el apilamiento vertical con nodos de destino que son de una versión del producto más antigua que la versión del gestor de despliegue, se inhabilita la adición de nuevos miembros en el mismo nodo.

Preferencias de aislamiento
Especifica una preferencia de aislamiento para el clúster dinámico. Una preferencia de aislamiento especifica si se desea que el clúster dinámico esté en ejecución en los mismos nodos que otras instancias de clústeres dinámicos, o si desea que el clúster dinámico sea el único clúster dinámico en ejecución en el nodo.
  • Seleccione No hay requisitos de aislamiento si las instancias de clúster en este clúster dinámico se pueden ejecutar en el mismo nodo que otras instancias de clústeres dinámicos distintos. Esta es la opción por omisión.
  • Seleccione Aislamiento estricto si instancias de clúster en este clúster dinámico sólo se pueden ejecutar con otras instancias del mismo clúster dinámico en el mismo nodo.
  • Seleccione Asociar con un grupo de aislamiento si la instancia de clúster se puede ejecutar con cualquier otra instancia de un clúster dinámico que esté en el mismo grupo compartido en el mismo nodo. Especifique un valor de Nombre de grupo de aislamiento. Escriba un nombre exclusivo para crear un nuevo grupo de aislamiento para este clúster dinámico. Pulse Examinar para seleccionar de una lista de grupos de aislamiento existentes. Se muestra una topología de los grupos de aislamiento actuales. Pulse sobre un nombre de un grupo de aislamiento para seleccionarlo.
Paso 4: Confirmar un nuevo clúster dinámico

Un resumen de las acciones realizadas muestra los cambios para la creación del clúster dinámico. Pulse Finalizar para crear el clúster dinámico.
