Valores de miembros de un clúster dinámico

Utilice esta página para gestionar los miembros de un clúster dinámico.

Para ver esta página de la consola administrativa, pulse Servidores > Clústeres dinámicos > nombre_clúster_dinámico > Miembros del clúster dinámico > nombre_miembro_clúster_dinámico .

Hay que poseer privilegios de operador para gestionar los miembros de un clúster dinámico. Lea el centro de información para obtener información sobre roles administrativos y privilegios para gestionar clústeres dinámicos.

Miembro

Especifica el nombre del servidor en el clúster. El nombre del servidor es el nombre del proceso. El nombre debe coincidir con el nombre de objeto del servidor de aplicaciones.

Tipo de datos: Serie

Peso

Especifica la cantidad de trabajo que se dirige al servidor de aplicaciones. Si el valor del peso del servidor es mayor que los valores de peso asignados a otros servidores en el clúster, el servidor recibe un parte mayor de la carga de trabajo del servidor.

El peso que se muestra depende de la pestaña en que se está mostrando la información. Si está viendo la pestaña Tiempo de ejecución, se muestra el peso de tiempo de ejecución. Si está viendo la pestaña Configuración, se muestra el peso configurado.
Importante: Si se habilita, el gestor de carga de trabajo dinámica (DWLM) altera temporalmente todos los valores que se entren. DWLM está habilitado por omisión. Para inhabilitar DWLM, pulse Servidores > Clústeres > Clústeres dinámicos > nombre_clúster_dinámico > Gestión de carga de trabajo dinámica (DWLM)
En la plataforma z/OS, el peso se utiliza para equilibrar algunos tipos de carga de trabajo, pero otros, los equilibra el sistema z/OS.
  • Para las solicitudes HTTP, se utilizan pesos para distribuir el tráfico HTTP entre el plug-in de servidor web y el controlador que maneja el servidor de aplicaciones en clúster. Asigne un valor de peso mayor al servidor de aplicaciones que está recibiendo el tráfico HTTP.
  • Para llamadas a servicios web, la información se transfiere de un sirviente de un servidor de aplicaciones a un controlador de otro servidor de aplicaciones. El servidor de aplicaciones que recibe la llamada tiene el valor de peso más alto.
  • El peso no afecta a las solicitudes IIOP (Internet Inter-ORB Protocol). Las solicitudes IIOP se distribuyen al servidor de aplicaciones correcto con el distribuidor de sysplex.

Tipo de datos: Entero

Rando: De 0 a 20

ID exclusivo

Especifica un identificador numérico para el servidor de aplicaciones que es exclusivo dentro del clúster. El ID se utiliza por motivos de afinidad. Después de que se establezca una sesión en un miembro concreto del clúster, las solicitudes posteriores que pertenecen a la misma sesión se asignan al miembro de clúster que ha establecido la afinidad de sesiones.

Tipo de datos: Entero

Tipo de datos: Hexadecimal
