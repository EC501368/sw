Paso 2: Seleccionar un método de pertenencia

El método de pertenencia define la forma en que se añaden las instancias de clúster al clúster dinámico. Se puede de crear una política de pertenencia para colocar instancias de clúster en determinados nodos o bien se pueden añadir de forma manual instancias de clúster al clúster dinámico.

Existe la posibilidad de definir un método de pertenencia para los servidores de aplicaciones, los servidores de WebSphere Application Server Community Edition y los servidores PHP. En todos los otros tipos de servidores se define la pertenencia de forma manual.

Para acceder a esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres dinámicos > Nuevo.

Definir automáticamente los miembros del clúster mediante normas

Con este método de pertenencia, se crea una política que define los servidores que se unen al clúster dinámico. Se puede definir una política para servidores con una gestión completa del ciclo de vida, incluidos servidores de aplicaciones, servidores de WebSphere Application Server Community Edition y servidores PHP. Si se crea un clúster dinámico PHP, se debe utilizar esta opción.

Nombre de clúster dinámico
Si está creando un clúster dinámico que está formado de servidores de aplicaciones, especifique un nombre para el clúster dinámico. El nombre de un clúster dinámico debe ser exclusivo entre todos los clústeres en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre no puede empezar con un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Preferencia local habilitada
Para habilitar o inhabilitar la optimización del direccionamiento con ámbito de nodo, seleccione Preferencia local habilitada. El valor predeterminado es habilitado, lo que indica que, si es posible, las peticiones de EJB (Enterprise JavaBeans) se direccionan al nodo cliente. Si habilita esta función, el rendimiento mejora porque las peticiones del cliente se envían a enterprise beans locales.
Crear un dominio de duplicación para este clúster
Especifica que se crea un dominio de duplicación para el clúster dinámico. El nombre del dominio de duplicación se establece en el nombre del clúster dinámico que utiliza este dominio. Los dominios de duplicación permiten transferir datos, objetos o sucesos para el gestor de sesiones, la antememoria dinámica o los beans de sesión sin estado entre los servidores de aplicaciones en un clúster.

Definir de forma manual a los miembros del clúster

Con este método de pertenencia, se definen de forma manual los servidores que se unen al clúster dinámico. Esta opción se utiliza de forma automática para servidores con una gestión asistida de ciclo de vida. Esta opción también permite convertir un clúster estático que ya exista en un clúster dinámico.
