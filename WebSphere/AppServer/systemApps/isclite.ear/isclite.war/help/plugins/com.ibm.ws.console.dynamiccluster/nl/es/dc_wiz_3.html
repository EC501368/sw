Paso 3: Definir los miembros del clúster dinámico

Al crear un clúster dinámico se pueden definir sus miembros, o añadirlos a un clúster dinámico que ya exista.

Para acceder a esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres dinámicos > Nuevo.

Para añadir miembros de clúster dinámicos a un clúster dinámico existente, pulse Servidores > Clústeres > Clústeres dinámicos > nombre_clúster_dinámico > Miembros del clúster dinámico > Añadir. Únicamente se pueden añadir miembros de clúster dinámico a clústeres dinámicos que estén en la modalidad de pertenencia manual y que no tengan definida una política de pertenencia.

Pertenencia basada en normas

Existe la posibilidad de utilizar una pertenencia basada en normas con los servidores de aplicaciones, los servidores de WebSphere Application Server Community Edition o los servidores PHP.

Utilice el generador de subexpresiones para crear una Política de pertenencia. La política de pertenencia especifica los nodos que son miembros del clúster dinámico. Pulse Previsualizar pertenencia para ver qué nodos están en el clúster dinámico con la política de pertenencia definida.

Políticas de pertenencia de ejemplo

  • Para definir la pertenencia de todos los nodos en un grupo de nodos específicos: 
    node_nodegroup = 'mi_grupo_nodos'
  • Para definir la pertenencia de todos los nodos en los que se ejecuta Apache.
    node_property$APACHE_2_0 IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta Apache y PHP:
    node_property$APACHE IS NOT NULL and node_property$PHP IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta WebSphere Application Server Community Edition Versión 2.0 o posterior:
    node_property$WASCE_2.0* IS NOT NULL
  • Para definir la pertenencia de todos los nodos en los que se ejecuta WebSphere Application Server Community Edition Versión 2.0 únicamente:
    node_property$WASCE_2.0 IS NOT NULL
  • Para definir la pertenencia de todos los nodos que tienen un nombre que es como mi_nodo, incluidos mi_nodo_1, mi_nodo_2, etc.:
    node_name like 'mi_nodo%'
  • Para definir la pertenencia de una lista específica de nombres de host:
    node_hostname in ('nodo1.midominio.com','nodo2.midominio.com')
Las subexpresiones se pueden agrupar con paréntesis y operadores lógicos and, or o not. En el siguiente ejemplo se define la pertenencia a un nodo como los nodos que no están en el grupo de nodos mi_grupo_nodos y tienen un nombre que es como mi_nodo, incluidos mi_nodo_1, mi_nodo_2, etc.
not (node_nodegroup='mi_grupo_nodos') and node_name like 'mi_nodo%'

Pertenencia definida manualmente

Utilice la pertenencia definida de forma manual para añadir servidores que existen en la configuración para añadirlos al clúster dinámico.

  • Con clústeres dinámicos que tienen servidores de aplicaciones, se puede utilizar un clúster estático que ya exista. Seleccione un clúster estático existente de la lista para utilizarlo.
  • Con servidores con una gestión asistida de ciclo de vida, seleccione los servidores que desee añadir al clúster dinámico. Pulse Añadir miembro. Los servidores que añada al clúster dinámico aparecerán en la lista. Si desea crear un nuevo miembro de clúster dinámico, pulse Añadir. Para eliminar un miembro y pulse Eliminar. Todos los miembros del clúster dinámico deben ser el mismo tipo de servidor.
