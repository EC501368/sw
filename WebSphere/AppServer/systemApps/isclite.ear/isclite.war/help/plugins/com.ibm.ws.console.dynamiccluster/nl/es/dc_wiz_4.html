Paso 4: Seleccionar una plantilla de servidor

Seleccione una plantilla de servidor para utilizar como modelo para los servidores para el nuevo clúster dinámico. Se puede seleccionar una plantilla por omisión, o correlacionar un servidor que ya exista.

Para acceder a esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres dinámicos > Nuevo.

Plantilla de servidor

Especifica la plantilla que es un modelo para los servidores en el clúster dinámico.

  • Seleccione Crear el miembro del clúster utilizando una plantilla de servidor para crear el miembro del clúster a partir de una plantilla de servidor definida. Seleccione una plantilla de servidor de la lista.
    En la siguiente lista se describen las plantillas para los servidores de aplicaciones:
    • Las plantillas default o defaultZOS son plantillas de WebSphere® Application Server Versión 6 o superior.
    • La plantilla DeveloperServer es la plantilla por omisión que se puede utilizar al desarrollar sus aplicaciones.

    También existe la posibilidad de especificar una plantilla para servidores de middleware con una gestión completa del ciclo de vida, como con servidores PHP y WebSphere Application Server Community Edition. Existen plantillas predefinidas que se pueden utilizar en los clústeres dinámicos o bien se pueden crear plantillas de servidor propias. Para crear su propia plantilla de servidor, pulse Servidores > Todos los servidores > Plantillas... > Nueva.

  • Seleccione Crear el miembro de clúster utilizando un servidor existente como plantilla para utilizar como modelo un servidor de aplicaciones existente para crear los servidores para el clúster dinámico. Seleccione un servidor de la lista.
  • Seleccione el grupo principal al que pertenece el miembro de clúster. Esta opción sólo está disponible cuando selecciona WebSphere Application Server como un tipo de servidor de clúster dinámico.

Las plantillas que se muestran corresponden a la plataforma y versión de producto del gestor de despliegue que esté utilizando.
