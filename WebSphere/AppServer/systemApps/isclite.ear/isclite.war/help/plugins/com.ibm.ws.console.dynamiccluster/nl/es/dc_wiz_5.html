Paso 5: Configurar las propiedades del clúster dinámico

Define las propiedades para el clúster dinámico.

Para acceder a esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres dinámicos > Nuevo.

Número mínimo de instancias de clúster

Especifica el número mínimo de instancias en ejecución para el clúster dinámico. Si un valor mínimo es excesivo, es posible que se produzca una degradación del rendimiento.

  • Si desea detener instancias cuando no haya actividad, pulse Detener todas las instancias durante periodos de inactividad. Si selecciona esta opción, indique el valor de Tiempo de espera antes de detener instancias en minutos.
    Restricción: si se está direccionando tráfico SIP (Session Initiation Protocol) al clúster dinámico, no utilice la opción Si otros clústeres dinámicos necesitan recursos, detener todas las instancias de este clúster en periodos de inactividad. Utilice una de las opciones que mantienen al menos activo un miembro del clúster todo el tiempo.
  • Si desea que siempre exista una instancia de clúster iniciada, pulse Mantener iniciada una instancia todo el tiempo. Esta es la opción predeterminada.
  • Si desea que siempre exista más de una instancia de clúster iniciada, pulse Mantener iniciadas varias instancias todo el tiempo. Si se selecciona esta opción, indique el Número de instancias que desea iniciadas todo el tiempo.

Número máximo de instancias de clúster

Especifica el número máximo de instancias de clúster que se pueden iniciar.

  • Si desea limitar el número de instancias de clúster, pulse Limitar el número de instancias que pueden iniciarse y especifique el Número de instancias máximo.
  • Si no desea limitar el número de instancias de clúster, pulse No limitar el número de instancias que pueden iniciarse.

Apilamiento vertical de instancias en el nodo

Especifica si se puede iniciar más de una instancia en un único nodo. Al configurar el apilamiento vertical, el controlador de ubicación de aplicaciones puede iniciar más de una instancia del clúster dinámico en el nodo cuando exista una carga de trabajo grande.

El apilamiento vertical sirve para mejorar situaciones de cuello de botella entre aplicaciones. Seleccione Permitir iniciar más de una instancia en el mismo nodo y escriba el valor de Número de instancias. El número de instancias es el número de servidores de aplicaciones para un clúster que se necesitan para utilizar todas las capacidades de un nodo. Además, el número que elija para el apilamiento vertical es el número de servidores que se crean automáticamente.

Si se utiliza el apilamiento vertical con nodos de destino que son de una versión del producto más antigua que la versión del gestor de despliegue, se inhabilita la adición de nuevos miembros en el mismo nodo. El apilamiento vertical únicamente está disponible en servidores de aplicaciones, servidores de WebSphere Application Server Community Edition y servidores PHP que tengan miembros de clúster que las normas definan automáticamente.

Preferencias de aislamiento

Especifica si se desea que el clúster dinámico esté en ejecución en los mismos nodos que otras instancias de clústeres dinámicos, o si desea que el clúster dinámico sea el único clúster dinámico en ejecución en el nodo. Esta opción únicamente está disponible si se elige definir de forma automática los miembros del clúster mediante normas. Si se definen los miembros del clúster de forma manual, no se tendrá la opción de habilitar el aislamiento de clúster dinámico.

  • Seleccione No hay requisitos de aislamiento si las instancias de clúster en este clúster dinámico se pueden ejecutar en el mismo nodo que otras instancias de clústeres dinámicos distintos. Esta es la opción predeterminada.
  • Seleccione Aislamiento estricto si instancias de clúster en este clúster dinámico sólo se pueden ejecutar con otras instancias del mismo clúster dinámico en el mismo nodo.
  • Seleccione Asociar con un grupo de aislamiento si la instancia de clúster se puede ejecutar con cualquier otra instancia de un clúster dinámico que esté en el mismo grupo compartido en el mismo nodo. Especifique un valor de Nombre de grupo de aislamiento. Escriba un nombre exclusivo para crear un nuevo grupo de aislamiento para este clúster dinámico. Pulse Examinar para seleccionar de una lista de grupos de aislamiento existentes. Se muestra una topología de los grupos de aislamiento actuales. Pulse sobre un nombre de un grupo de aislamiento para seleccionarlo.
