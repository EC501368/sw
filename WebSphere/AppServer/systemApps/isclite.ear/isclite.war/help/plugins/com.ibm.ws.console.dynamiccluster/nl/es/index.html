Clústeres dinámicos

Vaya a los temas de ayuda de esta sección.

  • Colección de clústeres dinámicos
    Utilice esta página para ver clústeres dinámicos que ya existen. Los miembros de los clústeres dinámicos, o instancias de servidor, residen en un nodo dentro de un grupo de nodos. Los clústeres dinámicos, así como sus grupos de nodos asociados, se utilizan para realizar la ubicación de aplicaciones cuando el entorno se está ejecutando en una modalidad automática.
  • Propiedades personalizadas del clúster dinámico
    Esta página sirve para crear o suprimir propiedades personalizadas. Para que las propiedades personalizadas funcionen correctamente, las parejas de nombre y valor han de tener alguna relevancia. Las propiedades personalizadas establecen propiedades de configuración del sistema interno, de forma que el establecerlas requiere un conocimiento significativo del producto.
  • Colección de miembros de clúster dinámico
    Utilice esta página para ver, iniciar y detener los miembros de un clúster dinámico.
  • Valores de miembros de un clúster dinámico
    Utilice esta página para gestionar los miembros de un clúster dinámico.
  • Valores de clúster dinámico
    Utilice esta página para ver configuraciones de clústeres dinámicos existentes.
  • Generador de políticas de pertenencia
    El generador de subexpresiones de políticas de pertenencia sirve para crear una expresión que define los miembros del clúster dinámico. La utilidad de generación de subexpresiones sirve para generar condiciones complejas de normas utilizando AND, OR y NOT así como la agrupación con paréntesis. El generador de normas valida la norma al aplicar los cambios y avisa si hay paréntesis desemparejados y operadores lógicos a los que no se les da soporte.
  • Paso 1: Seleccionar el tipo de servidor de clúster dinámico
    Este paso sirve para especificar el tipo de servidor y el nombre del nuevo clúster dinámico.
  • Paso 2: Seleccionar un método de pertenencia
    El método de pertenencia define la forma en que se añaden las instancias de clúster al clúster dinámico. Se puede crear una política de pertenencia para colocar automáticamente instancias de clúster en determinados nodos o bien se pueden añadir de forma manual instancias de clúster al clúster dinámico.
  • Paso 3: Definir los miembros del clúster dinámico
    Al crear un clúster dinámico se pueden definir sus miembros, o añadirlos a un clúster dinámico que ya exista.
  • Paso 4: Seleccionar una plantilla de servidor
    Seleccione una plantilla de servidor para utilizar como modelo para los servidores para el nuevo clúster dinámico. Se puede seleccionar una plantilla por omisión, o correlacionar un servidor que ya exista.
  • Paso 5: Configurar las propiedades del clúster dinámico
    Define las propiedades para el clúster dinámico.
  • Paso 6: Resumen
    Esta página de la consola administrativa sirve para ver y guardar valores al crear un clúster dinámico o añadir miembros de clúster dinámico. Esta página de resumen muestra los cambios de configuración antes de comprometerlos y antes de que se cree el nuevo clúster dinámico o que se añadan miembros de clúster dinámico.
