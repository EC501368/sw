Colección de acciones personalizadas

De forma predeterminada, se puede definir una política de salud para ejecutar cualquiera de las acciones de salud predefinidas, tal como reiniciar un servidor, cuando se ha incumplido una política de salud. Configure una acción personalizada cuando desee que el entorno realice una acción distinta a la especificada por omisión.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Acción personalizada.

Una acción personalizada es una definición de proceso. Se pueden entonces configurar las acciones personalizadas como una lista ordenada de pasos que se producen cuando se incumple una política de salud. Las acciones personalizadas se pueden ejecutar en los siguientes lugares de su entorno:
  • En el gestor de despliegue.
  • En cualquier nodo que aloje servidores que incumplen políticas de salud.
  • El nodo que ejecuta el supervisor de gestión de salud.
  • En un nodo especificado.

Nombre

Especifica un nombre para la acción personalizada.

Sistemas operativos soportados

Especifica los tipos de sistema operativo a los que la acción da soporte.

Acción

Específica el programa o el script que se ha de ejecutar para la acción. Se incluyen todos los argumentos que se tienen que pasar al archivo ejecutable.
