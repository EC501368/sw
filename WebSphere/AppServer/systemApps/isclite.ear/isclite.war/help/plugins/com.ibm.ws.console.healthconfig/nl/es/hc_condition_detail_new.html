Creación de una política de salud: definir las propiedades de la condición de salud

Utilice esta página para definir el comportamiento específico de la política de salud.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Políticas de salud > Nueva.

Los privilegios en relación a las políticas de salud dependen del rol administrativo del usuario. Los roles incluyen los del supervisor, el operador, el configurador y el administrador. Si está utilizando un rol de operador o supervisor, sólo puede ver la información de la política de salud. Si está utilizando un rol de configurador o administrador, tiene todos los privilegios de configuración en relación a las políticas de salud.

Propiedades de la condición de salud

Especifica las propiedades específicas para la condición de salud.

Tabla 1. Propiedades de condición basadas en la antigüedad
Valor Descripción
Antigüedad máxima

Este campo está disponible para la política basada en antigüedad. Los valores aceptables son números enteros positivos en días u horas entre 1 hora y 365 días. Para entrar un valor como 1,2 días, utilice 36 horas, puesto que no se da soporte a números decimales.
Tabla 2. Propiedades de condición de tiempo de espera de solicitud excesivo
Valor Descripción
Solicitudes con tiempo de espera excedido

Este campo está disponible para la condición de utilización de memoria excesiva. La condición de tiempo de espera de petición excesivo detecta, para cada servidor miembro de la política, el porcentaje de peticiones dirigidas al servidor cuyo tiempo de espera ha excedido (periodo de más de 60 segundos) después de ser direccionado desde el direccionador on demand. Los valores aceptables para este campo son números enteros entre 1 y 99.
Tabla 3. Propiedades de condición de tiempo de respuesta excesivo
Valor Descripción
Tiempo de respuesta

Este campo está disponible para la política de condición de tiempo de respuesta excesivo. La condición de tiempo de respuesta excesivo se infringe cuando el promedio del tiempo de respuesta de las solicitudes supera un intervalo de tiempo específico. Los valores aceptables para este campo están entre 1 milisegundo y 60 minutos.

Tabla 4. Condición de memoria: propiedades de uso de memoria excesivo
Valor Descripción
Tamaño de almacenamiento dinámico de JVM

La política de condición de exceso de utilización de memoria se infringe cuando el uso de memoria supera un porcentaje del tamaño del almacenamiento dinámico durante un periodo de tiempo. El porcentaje del total de memoria utilizada se combina con el valor de tiempo por encima del umbral para determinar cuándo se han de reiniciar los miembros. Los valores aceptables para este campo son números enteros entre 1 y 99.

Período de tiempo infractor

Este campo está disponible para la política de condición de utilización de memoria excesiva. La política de condición de exceso de utilización de memoria se infringe cuando el uso de memoria supera un porcentaje del tamaño del almacenamiento dinámico durante un periodo de tiempo. Los valores aceptables para este campo son entre 1 segundo y 60 minutos.

Tabla 5. Condición de memoria: propiedades de condición de pérdida de memoria
Valor Descripción
Nivel de detección
Se puede elegir entre los siguientes niveles de detección. En cada nivel hay un compromiso entre la velocidad y la precisión para detectar probables pérdidas de memoria.
  • Detección más rápida, mayor probabilidad de falsas alarmas: un nivel de detección más rápido detecta pérdidas de memoria potenciales con mayor rapidez, sin embargo, este nivel de detección tiene una mayor probabilidad de identificar falsamente una pérdida de memoria que una política de detección más lenta puesto que el análisis se realiza antes que el almacenamiento dinámico Java se amplíe a su tamaño configurado máximo.
  • Detección estándar, probabilidad estándar de falsas alarmas: un nivel de detección estándar es más preciso que uno más rápido, pero no es tan rápido a la hora de identificar una pérdida de memoria potencial. Los valores estándar y más rápido precisan la misma cantidad de datos históricos, sin embargo, el valor estándar realiza el análisis después de que el almacenamiento dinámico Java se haya ampliado a su tamaño configurado máximo.
  • Detección más lenta, menor probabilidad de falsas alarmas: un nivel de detección más lento es el más preciso, sin embargo, este nivel de detección no detecta una pérdida de memoria potencial tan rápidamente como lo hace el nivel de detección más rápido. Este valor más lento es el que necesita más datos históricos.
Tabla 6. Propiedades de la condición de reducción de saturación
Valor Descripción
Nivel de detección
  • Detección estándar, probabilidad normal de falsas alarmas: una política de detección estándar es menos precisa que una más lenta, pero más rápida para identificar una reducción de saturación potencial.

    Este nivel utiliza menos muestras (N=10) tanto para los tiempos de respuesta como para los pesos del gestor de carga de trabajo dinámica y detecta un punto de cambio en cada una de las métricas en base al conjunto de muestras. Como resultado, esta política llega a una conclusión con mayor rapidez puesto que espera a 20 muestras, 10 para la media izquierda y 10 para la media derecha, para calcular una diferencia de medias y buscar un máximo local. Las muestras se recogen a intervalos de 15 segundos. Por lo tanto, una reducción de saturación se detecta dentro de los 5 minutos de su aparición. Sin embargo, puesto que el número de muestras es menor, si las muestras tiene picos o descargadas transitorias, hay una probabilidad alta de falsas alarmas.

  • Detección más lenta, menor probabilidad de falsas alarmas: una política de detección más lenta es la más precisa, sin embargo, no detecta una reducción de saturación potencial con tanta rapidez que una política de detección estándar.

    Este nivel utiliza más muestras (N=15) tanto para los tiempos de respuesta como para los pesos del gestor de carga de trabajo dinámica. Como resultado, esta política llega a una conclusión con menor rapidez puesto que tiene que esperar 30 muestras (15 para la media izquierda y 15 para la media derecha) para calcular una diferencia de medias. El tiempo de detección es de siete minutos y treinta segundos. Sin embargo, puesto que se utilizan más muestras, la presencia de muestras con bajadas o picos transitorios no afecta tanto a los valores de las medias. Por lo tanto, la probabilidad de falsas alarmas es menor.

Tabla 7. Propiedades de condición de tiempo de carga de trabajo
Valor Descripción
Total de solicitudes

La política de condición de carga de trabajo se infringe cuando se ha dado servicio a un número determinado de peticiones definido por el usuario. Un valor de solicitud debe ser un número entero superior a 1000.
Tabla 8. Propiedades de la condición del porcentaje de recogida de basura
Valor Descripción
Porcentaje de tiempo empleado en la recogida de basura

La política de porcentaje de tiempo empleado en la recogida de basura supervisa una JVM (Java Virtual Machine) o un conjunto de JVM para determinar si está empleando un porcentaje de tiempo mayor en la recogida de basura durante un periodo de tiempo especificado. Las unidades son porcentajes. El valor predeterminado es 10. Los valores aceptables para este campo son números enteros entre 1 y 99.

Período de tiempo de muestreo

Este campo especifica el periodo de tiempo durante el que se recopilan los datos de recogida de basura. El porcentaje de tiempo empleado en la recogida de basura durante el periodo de muestreo debe estar por encima del valor del umbral antes de que se lleve a cabo la acción correctiva. Las unidades son minutos y horas. El valor predeterminado es de 2 minutos. Los valores aceptables para este campo están entre 1 minuto y 24 horas.

Tabla 9. Propiedades de condición personalizada
Valor Descripción
Ejecutar plan de reacción cuando Especifica una subexpresión que representa las métricas que se evalúan en la condición personalizada.

Reacción del supervisor de gestión de salud

Especifica cómo Intelligent Management se comporta cuando se alcanza una condición de salud definida.

Modalidad de reacción

Especifica la modalidad de reacción que define el comportamiento de la política de salud. La modalidad de reacción puede ser Supervisada o Automática.

  • Cuando una modalidad de reacción se establece en Supervisada, las políticas de seguridad están activas y las recomendaciones sobre las acciones se envían al administrador con una tarea de tiempo de ejecución. El administrador puede seguir las recomendaciones. Si el administrador aprueba una recomendación, de forma automática se toman las acciones para mejorar la condición de salud.
  • Cuando la modalidad de reacción se establece en Automática, las políticas de salud anotan cronológicamente datos de forma activa, y el producto de forma automática toma las acciones para mejorar las condiciones de salud, sin la aprobación del administrador.

Realizar las siguientes acciones cuando se incumple la condición de salud

Existe la posibilidad de definir un conjunto específico de acciones a realizar cuando se incumple la condición de salud. Estas acciones pueden ser acciones por omisión existentes, o pueden ser acciones personalizadas definidas por el usuario para ejecutar un archivo ejecutable.

Una lista de acciones muestra el orden en que se ejecutan cuando se incumple la condición de salud. Para añadir una acción, pulse Añadir acción.... Se puede elegir una acción de política de salud por omisión que ya exista, una acción personalizada que se haya creado, o se puede crear una nueva acción personalizada.

Para eliminar un paso, selecciónelo y pulse Acción Eliminar. Si desea cambiar el orden de los pasos, seleccione un paso y pulse Subir o Bajar.
