Generador de subexpresiones de condición de salud personalizada

El generador de subexpresiones de condición de salud permite definir una condición de salud personalizada para su política de salud. La utilidad de generación de subexpresiones sirve para generar condiciones complejas de normas utilizando AND, OR, NOT, así como la agrupación con paréntesis. El generador de subexpresiones valida la norma al aplicar los cambios y avisa si hay paréntesis desemparejados y operadores lógicos a los que no se les da soporte.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Políticas de salud > Nueva. Si se elige una condición de salud personalizada, se muestra el campo Ejecutar el plan de reacción cuando. Pulse Generador de subexpresiones para generar la condición de salud personalizada.

Seleccione las propiedades que desee incluir en su condición de salud y, a continuación, pulse Generar subexpresión. Aparecerá el valor de subexpresión. Si desea añadir la subexpresión a la condición de salud personalizada, pulse Agregar.

Operador lógico

Especifica el operador que se utilizará para agregar esta subexpresión a la subexpresión anterior en la condición de salud personalizada.

y
Ambas subexpresiones que el operador and relaciona deben ser ciertas para que se tengan en cuenta las acciones para la política de salud.
or
Si desea seleccionar una política de salud, una de las dos subexpresiones que el operador or relaciona deben ser ciertas para que se tengan en cuenta las acciones para la política de salud.

Operando

  • Métrica PMI: desde inicio servidor y Métrica PMI: desde último intervalo

    El operando Métrica PMI: desde inicio servidor utiliza un número medio de los valores recogidos desde el momento en que se inició el servidor.

    El operando Métrica de PMI: desde el último intervalo utiliza una media de los valores informados durante el último intervalo. El intervalo corresponde a la longitud del ciclo del controlador de salud.

    Ambos operandos PMI tienen los siguientes módulos PMI:

    • Módulo de agrupación de conexiones (JDBC): estas métricas únicamente se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server. El módulo de agrupación de conexiones tiene las siguientes métricas:
      • Anomalías: especifica el número de tiempos de espera de conexión excedidos en la agrupación.
      • Número de creaciones: especifica el número total de conexiones que se han creado.
      • Porcentaje utilizado: especifica el porcentaje medio de la agrupación que está en uso. Este valor se basa en el número total de conexiones configuradas en la agrupación de conexiones, no en el número actual de conexiones.
      • Descartes de antememoria de sentencias preparadas: especifica el número de sentencias que se descartan debido a que la antememoria está llena.
      • Número de destrucciones: especifica el número total de conexiones que se han cerrado.
      • Tamaño de agrupación: especifica el tamaño de la agrupación de conexiones.
      • Número de manejadores de conexión: especifica el número de objetos de conexión que se descartan debido a que la antememoria está llena.
      • Esperas concurrentes: especifica el número medio de hebras que están esperando una conexión al mismo tiempo.
      • Número de conexiones gestionadas: especifica el número de objetos ManagedConnection que están en uso para una agrupación de conexiones en concreto. Esta métrica únicamente se aplica a orígenes de datos de la versión 5.
      • Porcentaje con máximo: especifica el porcentaje medio del tiempo en que todas las conexiones están en uso.
      • Tiempo de JDBC (milisegundos): especifica el tiempo medio, en milisegundos, que se utiliza ejecutando llamadas JDBC (Java Database Connectivity). Este valor incluye el tiempo que se utiliza en la base de datos, la red y el controlador JDBC. Esta métrica únicamente se aplica a orígenes de datos de la versión 5.
      • Tiempo medio de utilización (milisegundos): especifica el tiempo medio en milisegundos en que se utiliza una conexión. El valor es el tiempo que transcurre desde que se asigna la conexión hasta que se devuelve. Este valor incluye el tiempo de operación de JDBC.
      • Número de devoluciones: especifica el número total de conexiones que se devuelven a la agrupación.
      • Tamaño de agrupación libre: especifica el número de conexiones libres en la agrupación.
      • Número de asignaciones: especifica el número de conexiones asignadas.
      • Tiempo medio de espera (milisegundos): especifica el tiempo medio de espera en milisegundos.
    • Módulo del sistema: Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo. El módulo de sistema tiene las siguientes métricas:
      • Utilización de CPU: desde inicio servidor: especifica la utilización media de la CPU desde que se inició el servidor.
      • Utilización de CPU: último intervalo: especifica la utilización media de la CPU desde la última consulta.
      • Memoria libre (MB): especifica una instantánea de la memoria libre en kbytes.
    • Módulo de proceso (para otros servidores): estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo. El módulo de proceso tiene las siguientes métricas:
      • Memoria residente de proceso (KB): especifica la memoria residente de proceso en Kbytes.
      • Utilización de CPU de proceso: desde inicio servidor: especifica la utilización de la CPU de proceso desde que se inició el servidor.
      • Utilización de CPU de proceso: último intervalo: especifica la utilización de la CPU de proceso en el último intervalo.
      • Memoria total de proceso (KB): especifica la memoria total de proceso en Kbytes.
    • Módulo EJB: estas métricas únicamente se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server. El módulo EJB tiene las siguientes métricas:
      • Media de métodos activos simultáneos: especifica el número medio de métodos que están activos al mismo tiempo.
      • Total llamadas de método: especifica el número de llamadas a métodos remotos del bean.
      • Submódulo de métodos: cargas de método: especifica la carga de método en el submódulo de métodos.
      • Almacenamientos: especifica el número de veces que los datos del bean se almacenaron en el almacenamiento persistente.
      • Número de mensajes: especifica el número de mensajes entregados al método onMessage del bean. Este número de mensajes se aplica a los beans controlados por mensajes.
      • Media de beans activos simultáneos: especifica la media de beans que están activos al mismo tiempo.
      • Eliminaciones: especifica el número de veces que se eliminaron los beans.
      • Devoluciones a agrupación: especifica el número de llamadas que están devolviendo un objeto a la agrupación.
      • En estado pasivo: especifica el número de veces que los beans se colocaron en el estado pasivo.
      • Obtenciones desde agrupación: especifica el número de llamadas que están recuperando un objeto desde la agrupación.
      • Drenajes desde agrupación: especifica el número de veces que el daemon encontró desocupada la agrupación e intentó limpiarla.
      • En estado de listo: especifica el número de instancias de bean que están en el estado de listo.
      • Tiempo medio de creación (milisegundos): especifica el tiempo medio, en milisegundos, para ejecutar una llamada de creación de bean. Este tiempo incluye el tiempo para cargar el bean.
      • Devoluciones descartadas: especifica el número de veces que el objeto devuelto se descartó porque la agrupación estaba llena.
      • Activaciones: especifica el número de veces que se activaron los beans.
      • Utilización sesión de servidor (porcentaje): especifica el porcentaje que se está utilizando la agrupación ServerSession. Esta métrica se aplica a beans controlados por mensajes.
      • Cargas: especifica el número de veces en que se cargaron los datos de bean desde el almacenamiento persistente.
      • Número mensajes retirados: especifica el número de mensajes retirados que no se pudieron entregar al método onMessage del bean. Esta métrica se aplica a beans controlados por mensajes.
      • Submódulo de métodos: tiempo de respuesta de método (milisegundos): especifica el tiempo de respuesta de método, en milisegundos.
      • Número en estado pasivo: especifica el número de beans que están en un estado pasivo.
      • Tamaño agrupación: especifica el número medio de objetos que se encuentran en la agrupación.
      • Tiempo de carga (milisegundos): especifica el tiempo medio en milisegundos para cargar los datos de bean desde el almacenamiento persistente.
      • Tiempo medio de eliminación (milisegundos): especifica el tiempo medio, en milisegundos, para ejecutar una llamada beanRemove. Este tiempo incluye el tiempo en la base de datos.
      • Obtenciones encontradas: especifica el número de veces en que una llamada de recuperación encontró un objeto disponible en la agrupación.
      • Tiempo de activación: especifica el tiempo medio, en milisegundos, para activar un objeto de bean.
      • Tamaño medio de drenaje: especifica el número medio de objetos que se descartaron en cada drenaje.
      • Submódulo de métodos: llamadas de método: especifica el número de llamadas de método.
      • Destrucciones: especifica el número de veces en que se destruyeron los beans.
      • Tiempo medio de espera de sesión de servidor (milisegundos): especifica el tiempo medio, en milisegundos, que se necesita para obtener una sesión de servidor desde la agrupación. Esta métrica se aplica a beans controlados por mensajes.
      • Creaciones: especifica el número de veces que se crearon los beans.
      • Tiempo medio de respuesta de método (milisegundos): especifica el tiempo de respuesta medio, en milisegundos, que transcurre para llamadas de método remotas en el bean.
      • Instancias: especifica el número de veces que se crearon instancias de bean.
      • Tiempo de almacenaje (milisegundos): especifica el tiempo, en milisegundos, para almacenar los datos de bean en el almacenamiento persistente.
      • Tiempo de estado pasivo (milisegundos): especifica el tiempo medio, en milisegundos, para que un objeto de bean se coloque en el estado pasivo.
    • Módulo de aplicación web: Estas métricas únicamente se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server. Los módulos de aplicación web tienen las siguientes métricas:
      • Número de errores: especifica el número total de veces que se recibió un error desde archivos JSP (JavaServer Pages) o servlets.
      • Total de solicitudes: especifica el número total de solicitudes que un servlet procesó.
      • Tiempo de respuesta (milisegundos): especifica el tiempo medio de respuesta, en milisegundos, en que se finalizan las solicitudes de servlet.
      • Solicitudes simultáneas: especifica el número de solicitudes que se están procesando en este momento.
      • Número de recargas: especifica el número de servlets que se han recargado.
      • Número de servlets cargados: especifica el número de servlets que se han cargado.
    • Módulo de tiempo de ejecución de JVM: estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo. Los módulos JVM (Java Virtual Machine) tienen las siguientes métricas:
      • Memoria libre (KB): especifica la memoria libre, en Kbytes, en el entorno de tiempo de ejecución de JVM.
      • Tiempo de actividad (segundos): especifica la cantidad de tiempo, en segundos, en que JVM ha estado en ejecución.
      • Total memoria (KB): especifica el total de memoria, en Kbytes, en el entorno de tiempo de ejecución de JVM.
      • Memoria utilizada (KB): especifica la cantidad de memoria utilizada, en Kbytes, en el entorno de tiempo de ejecución de JVM.
    • Módulo de agrupación de hebras: estas métricas únicamente se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server. Los módulos de agrupación de hebras tienen las siguientes métricas:
      • Número de detenciones de hebra: especifica el número de hebras que se han declarado como detenidas.
      • Porcentaje con máximo: especifica el porcentaje medio del tiempo en que todas las hebras están en uso.
      • Tiempo medio activo (milisegundos): especifica el tiempo medio en milisegundos en que las hebras están en un estado activo.
      • Destrucciones de hebra: especifica el número total de hebras que se han destruido.
      • Tamaño agrupación: especifica el número medio de hebras que hay en una agrupación.
      • Creaciones de hebra: especifica el número total de hebras que se han creado.
      • Hebras colgadas simultáneamente: especifica el número de hebras detenidas de forma simultánea.
      • Número de detenciones de hebra borradas: especifica el número de hebras detenidas que se han borrado.
      • Hebras activas: especifica el número de hebras que están activas de forma concurrente.
    Formato de subexpresión para Métrica PMI: desde inicio servidor:
    PMIMetric_FromServerStart$nombreMódulo$nombreMétrica operador valorLongL (con sufijo "L")
    Ejemplo: 
    PMIMetric_FromServerStart$systemModule$cpuUtilization  >  90L
    Formato de subexpresión para Métrica PMI: último intervalo notificado:
    PMIMetric_FromLastInterval$nombreMódulo$nombreMétrica operador valorLongL (con sufijo "L")
    Ejemplo: 
    PMIMetric_FromLastInterval$webAppModule$responseTime  > 200L
  • Métrica de nivel de servidor ODR: desde inicio servidor
    Se puede utilizar un subconjunto de las métricas de nivel de servidor que publica el direccionador On Demand (ODR). Estas métricas se acumulan y se recopilan a partir del momento en que se inicia el servidor.
    Nombre de métrica

    Se pueden utilizar las siguientes métricas de nivel de servidor.

    • Salidas: Especifica el número de solicitudes asignadas desde la cola al servidor durante el intervalo en que se informa. Se considera que una solicitud únicamente se asigna una vez, incluso si se produce un error con en el primer servidor y se vuelve a intentar en otro momento. El siguiente suceso después de asignar es devolver. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Tiempo de respuesta (milisegundos): especifica un tiempo de respuesta medio para las solicitudes. Para calcular esta media, calcula la suma de los tiempos de respuesta de las solicitudes que se devolvieron desde el servidor al cliente durante el intervalo en que se informa. La unidad de la suma es milisegundos. Divide la suma por el valor de la métrica serviced para obtener el tiempo de respuesta medio. El tiempo de respuesta de una solicitud es la suma del tiempo de espera y el tiempo de servicio de dicha solicitud. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Solicitudes en ejecución actualmente: especifica el número de solicitudes que están ejecución al final del intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Tiempo de servicio (milisegundos): especifica el tiempo medio de servicio de las solicitudes. Para calcular esta media, calcula la suma de cada tiempo de servicio de solicitud de las solicitudes que se devolvieron al servidor durante el intervalo en que se informa. La unidad de la suma es milisegundos. Lo divide por el valor de la métrica serviced en el mismo intervalo para obtener la media. El tiempo de servicio de una solicitud es el tiempo desde que se asigna hasta que se devuelve. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Tiempo de espera (milisegundos): especifica el tiempo medio de espera para las solicitudes. Para calcular esta media, calcula la suma del tiempo que cada solicitud está en espera en la cola durante el intervalo en que se informa. La unidad de la suma es milisegundos. Divide la suma por el número de salidas para obtener el tiempo de espera medio. Las solicitudes eliminadas no contribuyen a esta suma. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Errores: especifica el número de solicitudes que se han devuelto desde el servidor con un indicador de error durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Con servicio: especifica el número de solicitudes que se han devuelto desde el servidor al cliente durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Con tiempo de espera excedido: especifica el número de solicitudes que se han devuelto debido a un tiempo de espera de servicio excedido durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    Formato de subexpresión: 
    ODRServerMetric_FromServerStart$nombreMétrica operador valorLongL (con sufijo "L")
    Ejemplo: 
    ODRServerMetric_FromServerStart$errors  > 100L

Divide la suma por el número de salidas para obtener el tiempo de espera medio. Las solicitudes eliminadas no contribuyen a esta suma. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    Formato de subexpresión: 
    ODRCellMetric_FromServerStart$nombreMétrica operador valorLongL (con el sufijo "L")
    Ejemplo: 
    ODRCellMetric_FromServerStart$arrivals  > 10000L
  • Métrica de nivel de célula ODR: último intervalo notificado
    Se puede utilizar el mismo conjunto de métricas que con el operando de Métrica de nivel de célula ODR: desde inicio ODR. Esté operando utiliza una media de los valores recopilados del último intervalo. El intervalo corresponde a la longitud del ciclo del controlador de salud.
    • Salidas: Especifica el número de solicitudes asignadas desde la cola al servidor durante el intervalo en que se informa. Se considera que una solicitud únicamente se asigna una vez, incluso si se produce un error con en el primer servidor y se vuelve a intentar en otro momento. El siguiente suceso después de asignar es devolver. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Tiempo de respuesta (milisegundos): especifica un tiempo de respuesta medio para las solicitudes. Para calcular esta media, calcula la suma de los tiempos de respuesta de las solicitudes que se devolvieron desde el servidor al cliente durante el intervalo en que se informa. La unidad de la suma es milisegundos. Divide la suma por el valor de la métrica serviced para obtener el tiempo de respuesta medio. El tiempo de respuesta de una solicitud es la suma del tiempo de espera y el tiempo de servicio de dicha solicitud. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Longitud actual de cola: especifica la longitud de la cola al final del intervalo en que se informa.
    • Tiempo de servicio (milisegundos): especifica el tiempo medio de servicio de las solicitudes. Para calcular esta media, calcula la suma de cada tiempo de servicio de solicitud de las solicitudes que se devolvieron al servidor durante el intervalo en que se informa. La unidad de la suma es milisegundos. Lo divide por el valor de la métrica serviced en el mismo intervalo para obtener la media. El tiempo de servicio de una solicitud es el tiempo desde que se asigna hasta que se devuelve. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Errores: especifica el número de solicitudes que se han devuelto desde el servidor con un indicador de error durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Longitud media de cola: especifica la longitud media de la cola. Para calcular esta media, calcula la suma de las longitudes de cola que se notifican en cada llegada de solicitud antes de la inserción, y divide la suma por el número de llegadas. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Con servicio: especifica el número de solicitudes que se han devuelto desde el servidor al cliente durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Con tiempo de espera excedido: especifica el número de solicitudes que se han devuelto debido a un tiempo de espera de servicio excedido durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Solicitudes en ejecución actualmente: especifica el número de solicitudes que están ejecución al final del intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Llegadas: especifica el número de solicitudes que han llegado durante el intervalo en que se informa. El suceso siguiente, si hay, después de la llegada es asignar o eliminar. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Eliminaciones de desbordamiento de cola: especifica el número de peticiones que fueron aceptadas inicialmente en la cola en algún momento y después, en algún otro momento durante el intervalo en que se informa, fueron rechazadas de la cola. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Eliminaciones de cola: especifica el número de solicitudes que fueron aceptadas inicialmente en la cola en algún momento y que después fueron rechazadas de la cola en algún otro momento durante el intervalo en que se informa. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Retardados: especifica el número de solicitudes que han llegado durante el intervalo en que se informa y no fueron inmediatamente asignadas ni eliminadas. Estas métricas se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server u otros servidores de middleware con el agente de nodo.
    • Tiempo de espera (milisegundos): especifica el tiempo medio de espera para las solicitudes.   • Métrica de operación de MBean: tipo devuelto long y Métrica de operación de MBean: tipo devuelto string

    Para operandos de métricas de operación de beans gestionados (MBean), se debe especificar una serie de texto de consulta de nombre de objeto y el nombre del método del MBean. Estas métricas únicamente se pueden utilizar en servidores en los que se ejecuta WebSphere Application Server.

    Serie de texto de consulta de nombre de objeto
    Al crear la serie de texto de consulta de nombre de objeto, todos los caracteres especiales se deben codificar con el carácter barra invertida. El programa de fondo espera que la serie de texto de consulta del nombre de objeto esté en un formato en que todos los caracteres especiales se hayan codificado de forma apropiada.

    El valor que se entra para la serie de texto de consulta de nombre de objeto debe especificar las series de texto process=<nombre_proceso> y node=<nombre_nodo> o bien ninguna de ellas. Si se especifica process=<nombre_proceso> y node=<nombre_nodo>, el programa del fondo crea un sensor singleton MBeanSensor que detecta el MBean concreto en un servidor y nodo. Si no se especifica ninguno, el programa de fondo añade el nombre del servidor actual como nombre de proceso y el nombre del nodo actual como nombre de nodo, creando un sensor MBeanSensor por cada servidor al que se le aplique la política de salud. Si únicamente se especifica process=<nombre_proceso> o únicamente se especifica node=<nombre_nodo>, se producirá un error.

    Nombre de método de MBean
    Especifica el nombre del método del MBean a invocar.
    Métricas largas de formato de subexpresión: 
    MBeanOperationMetric_TypeLong$serieConsultaobjectName$nombreMétodo operador valorLongL (con sufijo "L")
    Ejemplo de métricas largas: 
    MBeanOperationMetric_TypeLong$WebSphere\:\*\,type\=HealthConditionLanguageInitializer\,node\=hipods3\,process\=nodeagent$getNumberOfOperands  > 10L
    Formato de subexpresión para métricas String: 
    MBeanOperationMetric_TypeString$serieConsultaobjectName$nombreMétodo operador valorSerie
    Ejemplo de métricas de serie: 
    MBeanOperationMetric_TypeString$WebSphere\:\*\,type\=HealthConditionLanguageInitializer\,node\=hipods3\,process\=nodeagent$getOperands  = 't'
  • Métrica de atributo de MBean: tipo devuelto long y Métrica de atributo de MBean: tipo devuelto string

  • Métrica de código de retorno de URL

    Con este operando, puede hacer ping a cualquier vía de acceso relativa (URI) en el servidor que es el destino de esta política. El valor devuelto se utiliza en la expresión de condición para la política de salud personalizada.

    Número de puerto de URL
    Especifique el número de puerto para realizar el ping.
    Vía de acceso relativa de URL
    Especifique el URL para realizar el ping. Todos los caracteres especiales en la serie se deben codificar con un carácter de barra invertida (\).
    Protocolo
    Especifica el protocolo a utilizar. Por ejemplo: http:// o https://
    Valor
    Especifica un entero corresponde al código de retorno del ping.

    Este operando se puede utilizar para hacer ping a cualquier URL de propósito general seleccionando el direccionador On Demand (ODR) como el destino de la política de salud y estableciendo las normas de direccionamiento apropiadas en el ODR.

    Este operando sirve para seleccionar miembros que ejecutan WebSphere Application Server o bien que ejecutan otro software de middleware con el agente de nodo.

    Formato de subexpresión: 
    URLReturnCodeMetric$númeroPuerto$víaAccesoRelativa operador valorInt
    Ejemplo: 
    URLReturnCodeMetric$9060$ibm\/console\/login\.do = 200
    El sensor de URL devuelve 0 si no es posible acceder al sitio web: 
    URLReturnCodeMetric$9060$ibm\/console\/login\.do = 0

Operador

  • Igual a (=): el operador de igualdad expresa una coincidencia sensible a las mayúsculas y minúsculas.
  • Distinto (<>): el operador distinto indica que el valor del operando no es igual al valor entrado.
  • Mayor que (>): el operador mayor que se utiliza con los números.
  • Mayor que o igual (>=): el operador mayor que o igual a se utiliza con números.
  • Menor que (<) : el operador menor que se utiliza con números.
  • Menor que o igual (<=): el operador menor que o igual a se utiliza con números.
  • Entre (BETWEEN): el valor debe estar entre el Límite inferior y Límite superior que se especifiquen.
  • En (IN): el valor debe estar en una lista de valores. Se pueden escribir valores y añadirlos a la lista.

Valor

Dependiendo del operador elegido, escriba un valor para la subexpresión que desea crear.

Subexpresión

Después de pulsar Generar subexpresión, este campo muestra el fragmento de la subexpresión generada en base a las opciones seleccionadas. Si desea añadir la subexpresión a la condición de salud personalizada, pulse Agregar.
