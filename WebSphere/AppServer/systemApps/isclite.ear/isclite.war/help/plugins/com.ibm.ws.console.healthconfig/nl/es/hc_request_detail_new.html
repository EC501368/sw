Definir propiedades de condición de tiempo de espera de petición excesivo

Utilice esta página para definir las propiedades de las condiciones de la política de salud cuando cree una nueva política de salud. Para ver esta página de la consola administrativa, pulse Políticas operativas > Políticas de salud > Nueva.

La siguiente es información ampliada sobre las propiedades de la política de salud:

Condición de tiempo de espera de petición excesivo:

Porcentaje de peticiones con tiempo de espera excedido para provocar el incumplimiento La política de condición de uso excesivo de memoria reinicia los miembros cuando el porcentaje peticiones que han superado el tiempo de espera para provocar el incumplimiento excede este valor. Los valores aceptables para este campo son números enteros entre 1 y 99.
Modalidad de reacción
  • Supervisar: indica que las políticas de salud están activas y las recomendaciones sobre las acciones adecuadas se envían al administrador, quien puede aceptarlas o rechazarlas.
  • Automático: indica que las políticas de salud están activas y que el sistema está anotando cronológicamente los datos y llevando a cabo las acciones.
Seleccionar acciones que deben llevarse a cabo al incumplir una condición de salud
  • Tomar vuelcos de hebras: toma vuelcos de las hebras únicamente en el Kit de desarrollo de Java de IBM.
  • Reiniciar servidor: reinicia el servidor.

Cuando haya completado los campos, pulse Siguiente.
