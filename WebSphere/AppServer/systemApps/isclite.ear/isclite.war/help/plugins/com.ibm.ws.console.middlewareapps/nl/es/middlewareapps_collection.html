Todas las aplicaciones

Utilice esta página para ver y gestionar todas las aplicaciones de su entorno. Las aplicaciones pueden incluir tanto aplicaciones Java EE (Java Platform, Enterprise Edition) como tipos de aplicación que se ejecutan en otros tipos de servidor.

Para ver esta página de la consola administrativa, pulse Aplicaciones > Todas las aplicaciones.

Si desea gestionar una aplicación de middleware instalada, habilite el recuadro de selección Seleccionar en la lista y pulse un botón:
Tabla 1. Botones de aplicación
Botón Acción resultante
Añadir... Se abre un asistente que ayuda a desplegar una aplicación o un módulo, como los archivos .jar, .zip , .tar, .ear, .war, .rar, tar.gz, tar.tgz o .gzip en un destino de despliegue. Existe la posibilidad de especificar scripts de configuración para ejecutarlos después que la aplicación esté instalada, o un script de limpieza para ejecutarlo después de la aplicación de desinstalación.

En el caso de servidores de perfiles Liberty, Tomcat, JBoss, Weblogic y Geronimo, se puede añadir una representación de la aplicación para el entorno administrativo. Hay que desplegar la aplicación fuera de la consola administrativa.
Eliminar Suprime la aplicación o la representación de la aplicación del depósito de configuración y suprime los binarios de la aplicación del sistema de archivos de todos los nodos en que están instalados los módulos de aplicación después de que la configuración esté guardada y sincronizada con los nodos.
Someter acción Realiza la acción seleccionada para una aplicación en concreto. Habilite el recuadro de selección Seleccionar en cada aplicación de middleware en que desee realizar la acción, seleccione la Acción que corresponda y pulse Someter acción.

Categoría Tipos de aplicación Descripción
Aplicaciones middleware gestionadas
Entre las aplicaciones gestionadas de middleware se incluyen:
  • PHP
  • WebSphere® Application Server Community Edition

Algunas aplicaciones de middleware, como las aplicaciones PHP, están formadas por un archivo archivador y scripts de configuración o limpieza opcionales. Entre los tipos de archivadores soportados se incluyen: .jar, .tar, .tgz, .tar.gz o .zip. Los scripts de configuración o limpieza están para ser ejecutados en cada servidor en que se despliega la aplicación. El script de configuración se ejecuta después de instalar la aplicación y antes de iniciar la aplicación. Se llama al script de limpieza al desinstalar la aplicación para eliminar todos los elementos residuales de la aplicación. Las acciones que se pueden incluir en un script incluyen el establecer permisos de archivos o creación y supresión de directorios.

Las aplicaciones de WebSphere Application Server Community Edition están formadas por planes de despliegue externos en lugar de scripts de configuración o limpieza. Entre los tipos de archivadores soportados se incluyen: .car, .ear , .jar, .raro .war. Un plan de despliegue externo es un artefacto de Apache Geronimo y WebSphere Application Server Community Edition. Estos planes de despliegue se definen de la misma manera que se definen los planes de despliegue en un entorno autónomo de WebSphere Application Server Community Edition.
Aplicaciones Java EE no gestionadas Aplicaciones Java EE que se ejecutan en los siguientes servidores:
  • Apache Tomcat
  • BEA WebLogic
  • Apache Geronimo
  • Servidor de perfiles Liberty
 El ODR puede direccionar peticiones a estos servidores, pero los gestores autónomos no gestionan en forma alguna los ciclos de vida de las aplicaciones Java EE no gestionadas. Sólo es posible añadir una representación de la aplicación en la consola administrativa. Hay que desplegar la aplicación fuera de la consola administrativa.
Aplicaciones Java EE gestionadas Aplicaciones Java EE que se ejecutan en servidores de aplicaciones WebSphere Application Server, incluidos los siguientes tipos de aplicaciones:
  • Cualquier aplicación Java EE típica.
 Estas aplicaciones se instalan con el asistente existente de instalación Java EE.

Estado

Indica si la aplicación desplegada en el servidor de middleware está iniciada, detenida o no está disponible.

Tabla 3. Estado
Icono Estado Descripción
Iniciado La aplicación está en ejecución.
Parcialmente iniciada La aplicación está en el proceso de cambiar de un estado de Detenida a un estado de Iniciada. Se está comenzando a ejecutar la aplicación pero todavía no lo está haciendo en su totalidad. La aplicación no se puede iniciar en su totalidad porque un servidor correlacionado con uno o más módulos de aplicación está detenido.
Detenido La aplicación no está en ejecución.
Parada parcial La aplicación está en el proceso de cambiar de un estado de Iniciada a un estado de Detenida. La aplicación todavía no se ha detenido.
No disponible No se puede determinar el estado.

Una aplicación con un estado de no disponible podría, de hecho, estar en ejecución pero tener un estado de no disponible porque el servidor que ejecuta la consola administrativa no se pudiese comunicar con el servidor en que la aplicación está en ejecución.
No aplicable La aplicación no proporciona información sobre si el servidor está en ejecución.
Únicamente para otras aplicaciones de middleware, la siguiente tabla indica el estado de despliegue general de una aplicación sólo cuando hay un problema.
Tabla 4. Estado de la aplicación de middleware
Icono Estado Descripción
Desplegado parcialmente La aplicación se ha desplegado en al menos uno de los servidores, sin embargo, como mínimo se desconoce el estado de uno de los servidores.
Error de despliegue parcial La aplicación no se desplegó en al menos uno de los servidores de forma satisfactoria.
Error de despliegue La aplicación no se desplegó en ninguno de los servidores de forma satisfactoria.
Despliegue desconocido No se puede determinar el estado en ninguno de los servidores.
