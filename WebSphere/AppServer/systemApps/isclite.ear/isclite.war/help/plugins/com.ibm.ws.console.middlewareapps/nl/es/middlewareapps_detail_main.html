Valores de una aplicación de middleware

Utilice esta página para ver configuraciones de aplicaciones de middleware que ya existan.

Para ver esta página de la consola administrativa, pulse Aplicaciones > Todas las aplicaciones > nombre_aplicación_middleware.

Nombre de aplicación

Especifica un nombre para la aplicación. El nombre de una aplicación de middleware debe ser único entre todas las aplicaciones en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre no puede empezar con un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

El nombre debe ser distinto de cualquier otro nombre de aplicación que se haya definido en la célula. Por ejemplo en una misma célula una aplicación web no gestionada no puede tener el mismo nombre que una aplicación Java EE (Java Platform, Enterprise Edition).

Tipo de aplicación

Especifica el tipo al que pertenece esta aplicación.

Edición de la aplicación

Especifica la edición de la aplicación.

Descripción de edición

Especifica la descripción de la edición de la aplicación.

Propiedades de despliegue

Especifica los módulos actuales y los lugares en que se encuentran desplegados en la configuración.

Se muestra una lista de los módulos definidos actualmente. Si desea cambiar un módulo que ya exista, selecciónelo y pulse Editar. Si desea eliminar un módulo que ya exista, selecciónelo y pulse Eliminar. Pulse Nuevo para añadir un módulo. Defina las propiedades para el módulo actual y pulse Aplicar.

Nombre de módulo

Especifica el nombre del módulo que se despliega como parte de la aplicación.

Raíz de contexto

Especifica la raíz de contexto de la aplicación web (WAR).

Host virtual

Especifica el nombre del host virtual en que está correlacionado actualmente el módulo web.

Si amplía la lista verá a una lista de host virtuales definidos con anterioridad. Seleccione un host virtual distinto de la lista para cambiar una correlación.

Destinos de despliegue

Especifica en qué lugar se despliega el módulo web. En caso de aplicaciones de WebSphere Application Community Edition, esta sección precede a la sección Propiedades de despliegue en el panel, y los destinos de despliegue se aplican a la aplicación en lugar de ser específicos de módulos individuales.

Se puede filtrar la lista de destinos de despliegue existentes ampliando la lista de Filtro y seleccionando la forma en que desea filtrar los destinos de despliegue. Pulse Añadir para añadir un destino de despliegue. El destino de despliegue que se añade se muestra en los destinos de despliegue de añadidos. Si desea eliminar un destino de despliegue, selecciónelo en los destinos de despliegue añadidos, y pulse Eliminar.
