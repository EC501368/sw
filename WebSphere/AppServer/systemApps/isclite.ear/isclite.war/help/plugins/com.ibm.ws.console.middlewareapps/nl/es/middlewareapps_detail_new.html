Creación de una nueva aplicación de middleware

Utilice estos paneles para instalar una nueva aplicación de middleware.

Para ver esta página de la consola administrativa, pulse Aplicaciones > Instalar una nueva aplicación de middleware.

Seleccione el tipo de aplicación

Aquí se especifica el tipo de aplicación que se añade.

Seleccione entre los siguientes tipos de aplicación:

  • Aplicaciones Java EE (Java Platform, Enterprise Edition)
  • Aplicaciones PHP (PHP Hypertext Preprocessor)
  • Aplicaciones Web no gestionadas
  • Aplicaciones WebSphere Application Server Community Edition

Subir el archivador de la aplicación

Especifica la vía de acceso completa de la aplicación que desea añadir.

El archivador de la aplicación se puede subir desde el sistema de archivos local o desde un sistema de archivos remoto. Los campos que hay que especificar dependen del tipo de aplicación que se instala.
Campos de aplicación PHP:
Nombre de vía de acceso completa
Especifica la vía de acceso al archivo de archivador. Los tipos que se pueden especificar son zip, tar, tar.gz, tar.tgz y jar.
Script de configuración
El script de configuración se ejecuta durante la instalación de la aplicación. Los scripts de configuración pueden incluir mandatos como los que establecen permisos de archivos y crean o suprimen directorios.
Script de limpieza
El script de limpieza se ejecuta al desinstalar la aplicación. Los scripts de limpieza pueden incluir mandatos como los que cambian permisos de archivos y crean o suprimen directorios.
Campos de aplicación de WebSphere Application Server Community Edition:
Archivador de aplicación
Ubicación remota o local del archivo de archivador de la aplicación. Los tipos que se pueden especificar son .car, .ear , .jar, .rar o .war.
Plan de despliegue externo
Ubicación remota o local del plan de despliegue.

Definir las propiedades generales

Se especifica el nombre, la edición y una descripción de edición de la aplicación que se desea añadir.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre no puede empezar con un punto (.) ni con un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Correlacionar aplicación con destinos de despliegue

Se especifican los destinos de despliegue para la aplicación. Este paso aparece al instalar aplicaciones de WebSphere Application Server Community Edition.

Correlacionar raíces de contexto y host virtuales para módulos web

Se especifica la ubicación de los host virtuales. Este paso aparece al instalar aplicaciones de WebSphere Application Server Community Edition.

Definir las opciones de despliegue

Define la información de despliegue para la aplicación. Este paso aparece al instalar aplicaciones de PHP.

  • Nombre de módulo: especifica el módulo web.
  • Raíz de contexto: especifica la raíz de contexto de la aplicación.
  • Host virtual: ubicación del host virtual.
  • Destino de despliegue: se puede filtrar según un tipo y obtener una lista de todos los destinos disponibles. Seleccione Añadir para mover la selección. Las opciones incluyen ninguno, sólo clústeres o servidores o por nombre.

Confirmar la nueva representación de la aplicación

Pulse Finalizar para completar la instalación de la aplicación de middleware.
