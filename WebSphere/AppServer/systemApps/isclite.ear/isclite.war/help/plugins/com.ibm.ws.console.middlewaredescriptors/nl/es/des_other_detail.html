Valores de detalle de versión

Utilice esta página para cambiar los valores específicos a una determinada versión de un descriptor de middleware. Por ejemplo, si se están ejecutando dos versiones distintas de PHP, se podrían cambiar estos valores para especificar ubicaciones de instalación para cada una de las versiones.

Para ver esta página de la consola administrativa, pulse Administración del sistema > Descriptores de middleware > tipo_plataforma_middleware > versión.

Versión

Especifica un nombre para la versión del descriptor de middleware.

Conector HTTP

Especifica las propiedades específicas del servidor, incluidos el número de puerto y la habilitación.

Según sea el tipo de servidor, se pueden mostrar otros campos. Por ejemplo, los servidores PHP tienen campos para PHP. HTPP no se muestra para todos los tipos de servidor.

Conector HTTPS

Especifica las propiedades específicas del servidor, incluidos el número de puerto, la habilitación y la seguridad.

Según sea el tipo de servidor, se pueden mostrar otros campos. HTPPS no se muestra para todos los tipos de servidor.

Ubicación raíz de servidor Web Apache

Si se seleccionó la versión del descriptor de middleware para el servidor Apache, se pueden cambiar estos valores para especificar dónde está instalado el servidor de Apache.

Ubicación de instalación de Windows
Especifica la vía de acceso en que Apache está instalado en un sistema operativo Windows. Se pueden listar varias ubicaciones separando las entradas de la lista con un punto y coma (;).
Ubicación de instalación de UNIX
Especifica la vía de acceso en que Apache está instalado en un sistema operativo como Linux o AIX.

Ubicación raíz del servidor PHP

Si se seleccionó la versión del descriptor de middleware para el servidor PHP, se pueden cambiar estos valores para especificar dónde está instalado servidor de PHP.

Ubicación de instalación de Windows
Especifica la vía de acceso en que PHP está instalado en un sistema operativo Windows. Se pueden listar varias ubicaciones separando las entradas de la lista con un punto y coma (;).
Ubicación de instalación de UNIX
Especifica la vía de acceso en que PHP está instalado en un sistema operativo como Linux o AIX.

Ubicación raíz de WebSphere Application Server Community Edition

Si la versión del descriptor de middleware que ha seleccionado era un servidor WebSphere® Application Server Community Edition, se pueden cambiar estos valores para especificar dónde está instalado el servidor WebSphere Application Server Community Edition.

Ubicación de instalación de Windows
Especifica la vía de acceso en que WebSphere Application Server Community Edition está instalado en un servidor Windows. Se pueden listar varias ubicaciones separando las entradas de la lista con un punto y coma (;).
Ubicación de instalación de UNIX
Especifica la vía de acceso en que WebSphere Application Server Community Edition está instalado en sistemas operativos como Linux o AIX.

Descriptor de afinidad de sesión

El descriptor de afinidad de sesión se utiliza cuando el direccionador On Demand (ODR) procesa una petición.

Aprender de ID de clon
Si se especifica como verdadero, el ORD analiza identificadores de clon de las cabeceras Set-Cookie de las respuestas a medida que el flujo de respuestas vuelve al servlet. Establezca esta propiedad en true cuando el servidor al cual esté direccionando no sea un servidor de aplicaciones WebSphere Application Server, y el valor de la cookie pueda analizarse para aprender el ID de clon.
Nombres de cookies
Especifica una lista de nombres de cookies separados por comas que se pueden utilizar para pasar información de sesión.
Reescribir URL
Especifica una lista de nombres separados por comas para buscar en el localizador uniforme de recursos (URL) información de sesión.
Separador de ID de clon
Especifica el delimitador que ODR utiliza al extraer ID de clon de la información de sesión.
Separador de ID de clon alternativo
Especifica un delimitador opcional para servidores que dan soporte a más de un carácter delimitador.
Modalidad de afinidad
Especifica la modalidad de afinidad para el tipo de plataforma de middleware. Este valor se utiliza junto con el valor Aprender de ID de clon para determinar cómo se utilizan los descriptores de afinidad de sesión.
  • Modalidad de afinidad pasiva: la información de clon de servidor se obtiene de la cookie JSESSIONID.
  • Con una modalidad de afinidad activa-condicional y un valor de Aprender de ID de clon de falso, la identificación de clon de servidor se genera de forma interna y se establece en la cookie WSJSESSIONID.

Tiempos de espera de servidor

Especifica el tiempo que debe pasar antes que se considere que se ha excedido el tiempo de espera de una operación de inicio o detención.

Tiempo de espera de inicio
Especifica el número de milisegundos antes que se considere que se ha excedido el tiempo de espera de una operación de inicio.
Tiempo de espera de detención
Especifica el número de milisegundos antes que se considere que se ha excedido el tiempo de espera de una operación de detención.
