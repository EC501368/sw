Colección de nodos de middleware

Los nodos de middleware incluyen a todos los nodos de su configuración. Estos nodos pueden incluir nodos que tienen instalada una versión del producto nodos que están ejecutando otros tipos de servidores.

Para ver esta página de la consola administrativa, pulse Administración del sistema > Nodos de middleware.

Tabla 1. Acciones de nodos de middleware
Acción Qué puede realizar con la acción
Seleccione una acción administrativa y pulse el botón Realizar. Realizar acciones administrativas:
Añadir un nodo
Añade un nodo al entorno administrativo.
Eliminar un nodo
Suprime los nodos seleccionados. Si no se elimina el nodo con esta acción, utilice la acción Suprimir un nodo.
Suprimir un nodo
Fuerza la eliminación de un nodo que no se ha eliminado correctamente de la célula en el depósito maestro. La acción Eliminar un nodo es más adecuada que la acción Suprimir un nodo para suprimir un nodo de la configuración.
Sincronizar un nodo
Solicita el realizar la operación de sincronización de nodo con el algoritmo de optimización de sincronización normal. Esta operación es rápida, pero quizás no corrija problemas de ediciones de archivo manuales que se realizan en el nodo. Es posible que la configuración de la célula y el nodo sigan sin estar sincronizadas después de haber realizado esta operación.
Sincronización completa
Borra todos los valores de optimización de sincronización y realiza una sincronización de configuración, de forma que hay una coincidencia entre la configuración de la célula y el nodo después de realizar esta operación. Esta operación puede tardar más que la operación Sincronizar.
Selección una acción operativa y pulse el botón Ejecutar. Realizar estas acciones operativas en los nodos de middleware que seleccione:
Detener agente
Detiene el proceso del agente en los nodos seleccionados. El estado de estos nodos debe ser Iniciado.
Iniciar agente
Inicia el proceso del agente en los nodos seleccionados.
Reiniciar agente
Reinicia el proceso del agente en los nodos seleccionados.
Reiniciar todos los servidores
Reinicia todos los servidores que hay en los nodos seleccionados.
Ejecutar descubrimiento
Fuerza la ejecución de todos los sensores. Los sensores pueden descubrir entornos de tiempo de ejecución Apache WebSphere® Application Server Community Edition, y PHP.
Seleccione una modalidad y pulse Establecer modalidad. Cuando se establece la modalidad de mantenimiento, existe la posibilidad de realizar diagnósticos, mantenimiento o ajustes en el nodo sin interrumpir el tráfico de entrada al servidor en un entorno de producción.
Mantenimiento
Coloca en la modalidad de mantenimiento después de que se haya dado servicio a las solicitudes que restan en el nodo.
Detención inmediata de mantenimiento
Coloca al nodo en la modalidad de mantenimiento sin dar servido a las solicitudes que restan en el nodo.
Normal
Saca al nodo de la modalidad de mantenimiento, de forma que los servidores en el nodo pueden comenzar a dar servicio a las solicitudes provenientes del direccionador On Demand.

Nombre

Especifica un nombre para un nodo que es exclusivo dentro de la célula.

Un nodo se corresponde con un sistema físico con una dirección IP distinta de host. Normalmente, el nombre de nodo es el mismo que el nombre de host del sistema.

Versión

Especifica la versión del producto del nodo.

La versión del producto corresponde a una versión de WebSphere Application Server.

Valor de versión Nombre de producto
Base WebSphere Application Server
Express® WebSphere Application Server Express
ND WebSphere Application Server Network Deployment

Sincronización

Especifica si los archivos de configuración en este nodo de middleware están sincronizados con el gestor de despliegue.

Si desea sincronizar nodos, seleccione los nodos que desee sincronizar. Elija la opción Sincronizar o Sincronización completa de las acciones administrativas y, a continuación pulse Realizar.

Tabla 3. Sincronización del nodo de middleware
Icono Acción Descripción
Sincronizado Los archivos de configuración en este nodo están sincronizados con el gestor de despliegue.
No sincronizado Los archivos de configuración en este nodo no están sincronizados con el gestor de despliegue y están obsoletos. Efectúe una operación de sincronización para obtener los últimos cambios de configuración en el nodo.
Desconocida No se puede determinar el estado del archivo de configuración porque no se puede acceder al agente de nodo para este nodo.
No aplicable La columna de estado es No aplicable a este nodo porque se trata de un nodo no gestionado.

Estado

Indica el estado del agente que está en ejecución en el nodo.

Tabla 4. Estado del nodo de middleware
Icono Acción Descripción
Iniciado El nodo está en ejecución.
Detenido El nodo no está en ejecución.
Desconocida El estado del nodo no está disponible.
No aplicable El estado del nodo es No aplicable porque no se ha informado del mismo al gestor de despliegue.

Modalidad de mantenimiento

Especifica el estado de la modalidad de mantenimiento del nodo.

Tabla 5. Modalidad de mantenimiento de nodos de middleware
Icono Acción Descripción
Modalidad de mantenimiento El nodo está en modalidad de mantenimiento y, por tanto, ya no está dando servicio a las solicitudes, de forma que es posible realizar acciones administrativas en este nodo. Mientras está en esta modalidad, se puede iniciar o detener el nodo. Cuando el nodo está iniciado y en la modalidad de mantenimiento, el direccionador On Demand no direcciona tráfico SIP o HTTP al nodo.
Modalidad normal El nodo está dando servicio a solicitudes.
