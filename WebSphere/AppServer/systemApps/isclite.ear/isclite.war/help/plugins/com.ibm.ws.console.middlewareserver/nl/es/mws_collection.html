Colección de servidores de middleware

Utilice esta página para ver y gestionar servidores de middleware. Los servidores de middleware son servidores genéricos en el nivel de middleware que alojan aplicaciones o sus datos.

Utilice esta página para crear nuevos servidores de middleware, plantillas de servidor de middleware o suprimir servidores de middleware que ya existan.

Para ver esta página de la consola administrativa, pulse Servidores > Todos los servidores o seleccioneServidores > Tipos de servidor y luego seleccione un tipo específico de servidor de middleware. Utilice el icono de filtro compuesto para formar vistas de colecciones específicas según un tipo.
Tabla 1. Botones de servidor de middleware
Botón Acción resultante
Nueva Abre un asistente que ayuda a crear un servidor de middleware. Los tipos de servidor incluyen servidor de aplicaciones, genérico, direccionador On Demand, proxy o servidor web. Cada tipo de servidor tiene un asistente exclusivo. También se puede añadir un servidor que ya exista en su entorno que no ejecute WebSphere®, tal como un servidor PHP, un servidor HTTP personalizado, un servidor WebSphere Application Server Community Edition o un servidor de perfiles Liberty.
Suprimir Elimina el servidor.
Plantillas Abre la lista de plantillas de servidor que hay definidas. Las plantillas de servidor son copias de datos de configuración que sirven como punto de partida para crear servidores. Las plantillas se pueden crear o editar si ya existen.
Iniciar Inicia uno o más servidores que seleccione. Habilite el recuadro de selección Seleccionar para cada servidor de middleware que desee iniciar y, a continuación, pulse Iniciar.
Detener Detiene uno o más servidores que seleccione. Habilite el recuadro de selección Seleccionar para cada servidor de middleware que desee detener y, a continuación, pulse Detener.
Terminar Termina de forma inmediata un servidor.
Someter acción Ejecuta la lista de acciones para un servidor web en la columna de acciones.
Establecer modalidad Establece la modalidad de mantenimiento en los servidores seleccionados. Elija una modalidad de mantenimiento y pulse Seleccionar modalidad. Cuando se establece la modalidad de mantenimiento, existe la posibilidad de realizar diagnósticos, mantenimiento o ajustes en el servidor sin interrumpir el tráfico de entrada al servidor en un entorno de producción.
Modalidad de mantenimiento
Da servicio a las solicitudes que quedan en el servidor. Todas las solicitudes que tengan una sesión abierta en el servidor se direccionan al servidor hasta que la sesión finaliza o excede el tiempo de espera. Una vez que todas las solicitudes se han completado, el servidor se coloca en modalidad de mantenimiento. Todas las nuevas solicitudes se direccionan a servidores que no están en modalidad de mantenimiento.
Modalidad de mantenimiento - interrumpir afinidad
Coloca a los servidores seleccionados en la modalidad de mantenimiento, pero interrumpe la afinidad de solicitudes SIP y HTTP para los servidores.
Detención de mantenimiento - detención inmediata
Coloca inmediatamente el servidor en modalidad de mantenimiento, sin atender las solicitudes que quedan en el servidor.
Normal
Inhabilita la modalidad de mantenimiento en los servidores seleccionados.

Nombre

Especifica un nombre para el servidor de middleware. El nombre de un servidor de middleware debe ser exclusivo en la célula.

Nodo

Nodo en que está el servidor de middleware.

Las acciones en el menú desplegable dependen del tipo de servidor. Por ejemplo, Generar y propagar plugin únicamente es válido en servidores web.

Nombre de clúster

Especifica el nombre del clúster al que pertenece el servidor.

Estado

Indica si el servidor de middleware está iniciado, detenido o si no está disponible.

Tabla 2. Estado del servidor de middleware
Icono Acción Descripción
Iniciado El servidor está en ejecución.
Parcialmente iniciada El servidor está en el proceso de cambiar de un estado de Detenido a un estado de Iniciado. Se está comenzando a ejecutar el servidor pero todavía no lo está haciendo en su totalidad.
Detenido El servidor no está en ejecución.
Parada parcial El servidor está en el proceso de cambiar de un estado de Iniciado a un estado de Detenido.
No disponible No se puede determinar el estado.
No aplicable El servidor no proporciona información sobre si está en ejecución.

Modalidad de mantenimiento

Especifica el estado de la modalidad de mantenimiento del servidor.

Tabla 3. Estado de la modalidad de mantenimiento del servidor de middleware
Icono Acción Descripción
Modalidad de mantenimiento El servidor está en modalidad de mantenimiento y, por tanto, ya no está dando servicio a las solicitudes, de forma que es posible realizar acciones administrativas en este servidor. Mientras está en esta modalidad, se puede iniciar o detener el servidor. Cuando el nodo está iniciado y en la modalidad de mantenimiento, el direccionador On Demand no direcciona tráfico SIP o HTTP al servidor.
Modalidad normal El servidor está dando servicio a solicitudes.

Versión

Especifica el número de versión de WebSphere Network Deployment.

Tipo

Especifica el tipo de servidor de middleware. Las opciones incluyen servidor de aplicaciones, servidor genérico, direccionador On Demand, servidor proxy, servidor apache y servidor web. Entre los otros tipos de servidor se incluyen los servidores PHP, Apache Tomcat, BEA Weblogic, JBoss, WebSphere Application Server Community Edition, servidores de aplicaciones WebSphere externos, servidores de perfiles Liberty, servidores WebSphere MQ y HTTP personalizado.
