Creación de un servidor de middleware direccionador On Demand

Esta página sirve para crear un nuevo servidor de middleware direccionador On Demand.

Pulse Servidores > ODR > Nuevo o bien pulse Servidores > Otros servidores de middleware > Seleccionar un tipo de servidor > Nuevo para ver esta página de la consola administrativa.

Hay que poseer privilegios de administrador para crear un servidor de middleware direccionador On Demand. Consulte la sección de referencia en el centro de información para obtener información sobre otros roles administrativos y privilegios.

Crear una nueva instancia de servidor

Este panel del asistente sirve para crear una nueva instancia de servidor para el entorno de WebSphere Extended Deployment.

Paso 1: Añadir un servidor

Crea un nuevo servidor o añade un servidor no WebSphere Application Server que ya existe.

Crea una nueva instancia de servidor que es exclusiva en la célula.
Elija un servidor direccionador On Demand de la lista:
  • Aplicación
  • Genérico
  • Direccionador On Demand
  • PHP
  • Proxy
  • Web
Añadir un servidor existente en la célula:
Elija un tipo de servidor de la lista:
  • CustomHTTP
  • BEA Weblogic Server
  • JBoss
  • PHP
  • Apache Tomcat
  • WASCE
Paso 2: Seleccionar un nodo

Seleccione un nodo que corresponda al servidor que desea crear y defina un nombre de servidor.

Paso 3: Especificar propiedades específicas del servidor

Seleccione los recuadros de selección de protocolos y Generar puertos únicos que desee asignar al servidor direccionador On Demand.

Paso 2: Seleccionar una plantilla de servidor

Seleccione la plantilla que especifique mejor los atributos del servidor que se está creando.

Paso 5: Confirmar el nuevo servidor direccionador On Demand

El campo de resumen de acciones describe las elecciones realizadas con el asistente. Pulse Finalizar para crear el nuevo servidor direccionador On Demand.
