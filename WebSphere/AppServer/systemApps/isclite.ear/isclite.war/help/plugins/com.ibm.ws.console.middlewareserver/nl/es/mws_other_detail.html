Valores de detalle de servidor de middleware

Esta página sirve para ver la configuración de los servidores de middleware.

Para ver esta página de la consola administrativa, pulse Servidores > Todos los servidores > servidor_middleware, o seleccione Servidores > Tipos de servidor > tipo_servidor_middleware > servidor_middleware.

Nombre

Especifica un nombre para el servidor. El nombre para un servidor debe ser exclusivo en la célula.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % ' . El nombre definido debe ser exclusivo entre grupos de servidores y no puede comenzar por un punto (.) o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Nombre del nodo

Especifica el nombre de nodo asociado con el servidor. Normalmente, un nombre de nodo es el mismo que el nombre de host del sistema. Sin embargo, existe la posibilidad de elegirlo. Se puede hacer que el nombre de nodo sea distinto del nombre de host.

Conector HTTP

Especifica las propiedades específicas del servidor, incluidos el número de puerto y la habilitación.

Según sea el tipo de servidor, se pueden mostrar otros campos. Por ejemplo, los servidores PHP tienen campos para PHP. Los conectores de HTPP y HTTPS no se muestran con todos los tipos de servidor.

Conector HTTPS

Especifica las propiedades específicas del servidor, incluidos el número de puerto, la habilitación y la seguridad.

Los conectores de HTPP y HTTPS no se muestran con todos los tipos de servidor.

Configuración del servidor

Especifica propiedades adicionales de los servidores de WebSphere Application Server Community Edition.

Depósito de aplicaciones
Especifica la ubicación del depósito de aplicaciones que utiliza el servidor de WebSphere Application Server Community Edition. En esta vía de acceso se pueden especificar variables como, por ejemplo, ${GERONIMO_HOME} para indicar la primera parte de la vía de acceso de la instalación. Para crear nuevas variables y modificar variables que ya existan, pulse Entorno > Variables de WebSphere.
Carpeta de configuración de servidor
Especifica la carpeta que contiene los archivos de configuración para el servidor. En esta vía de acceso se pueden especificar variables como, por ejemplo, ${GERONIMO_HOME} para indicar la primera parte de la vía de acceso de la instalación. Para crear nuevas variables y modificar variables que ya existan, pulse Entorno > Variables de WebSphere.
Versión WebSphere Application Server Community Edition
Especifica la versión de WebSphere Application Server Community Edition que está en ejecución en el servidor. Especifique el número de versión con puntos como, por ejemplo, 2.0.
Desplazamiento de puerto
Especifique un desplazamiento de puerto para evitar conflictos de puertos. El desplazamiento de puerto se utiliza al apilar servidores de WebSphere Application Community Edition e incrementa los valores de puerto. Por ejemplo, para la instancia de servidor por omisión, los valores de puerto podrían ser 8085 para HTTP y 8443 para HTTPS. Si se especifica un desplazamiento de puerto de 5, cuando se cree un nuevo servidor, el valor del puerto HTTP será 8090 y el valor del puerto HTTPS será 8448. De crearse un tercer servidor, entonces los valores de puerto serían 8095 para HTTP y 8453 para HTTPS.
La seguridad JMX está habilitada
Especifica si la seguridad de JMX (Java Management Extensions) está habilitada en un servidor de WebSphere Application Server Community Edition Versión 2.1 o posterior.
