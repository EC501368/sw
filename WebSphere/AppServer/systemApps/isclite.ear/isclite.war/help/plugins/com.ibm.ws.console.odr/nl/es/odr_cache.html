Valores de normas de antememoria estática del direccionador On demand

En este tema se describe cómo configurar una norma de antememoria asociada a un grupo de URI para el direccionador On Demand (ODR). HTTP 1.1 define un conjunto de normas para la colocación en antememoria del contenido en los direccionadores ODR. Las normas de antememoria estática permiten alterar temporalmente estas normas por omisión para un espacio de direcciones dado. Para que las normas tengan sentido, debe habilitar la memoria caché en la página de la consola administrativa Servidores > Tipos de servidores > Direccionadores On Demand > nombre_ODR > Propiedades del direccionador On Demand > Valores del direccionador On Demand.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidores > Direccionadores On Demand > nombre_ODR > Propiedades del direccionador On Demand > Reglas de memoria caché estática > grupo_URI.

Hay que poseer privilegios administrativos de configurador o administrador para cambiar los valores de ODR.

Puede editar los campos de valores de ODR en la pestaña Configuración.

Grupos URI

Los grupos de URI, junto con el host virtual, definen el ámbito del espacio de direcciones para que se realice la personalización de la memoria caché.

El campo de nombre no puede contener los caracteres siguientes: # \ / , : ; " * ? < > | = + & % '

El nombre definido debe ser exclusivo entre grupos URI y no puede comenzar por un punto o un espacio. Un espacio no genera un error, pero los espacios al principio o al final se eliminan automáticamente.

Inhabilitar la antememoria para este grupo URI

Inhabilita la colocación en antememoria para este espacio de direcciones. Es posible que un usuario desee inhabilitar la colocación en antememoria para un conjunto de servidores que se sabe que contienen información delicada o altamente personalizada.

El valor predeterminado es false, que indica que la antememoria está habilitada para el grupo URI.

Caducidad por omisión

El valor de caducidad por omisión, en segundos, que se utiliza para determinar la validez de una respuesta colocada en antememoria cuando las demás cabeceras de respuesta relacionadas con la antememoria de HTTP 1.1 no facilitan ninguna orientación. El valor predeterminado es suficiente en la mayoría de los entornos.

Último factor modificado

El porcentaje de una cabecera que se ha modificado por última vez para una respuesta que determina la validez de una respuesta colocada en antememoria cuando la respuesta no tiene cabeceras de caducidad HTTP explícitas. El valor predeterminado es suficiente en la mayoría de los entornos.

El valor predeterminado para este campo es 0.0.

Nombre del sistema principal virtual

El nombre del host virtual, junto con el grupo de URI, define el espacio de direcciones para el que debe realizarse la personalización de la antememoria.
