Colección de instancias de antememoria de objetos del direccionador On demand

Utilice esta página para configurar y gestionar instancias de memoria caché de objetos que, además de la memoria caché compartida predeterminada, pueden almacenar, distribuir y compartir datos de aplicaciones Java EE (Java Platform, Enterprise Edition). Utilice estas instancias de antememoria para proporcionar a las aplicaciones mejor flexibilidad y ajuste de los recursos de antememoria.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidores > Direccionadores On Demand > nombre_ODR > Propiedades del direccionador On Demand > Configuración de la instancia de memoria caché del direccionador On-Demand.

Hay que poseer privilegios administrativos de configurador o administrador para cambiar los valores del direccionador On Demand (ODR).

Utilice la interfaz de programas DistributedObjectCache para acceder a las instancias de antememoria. Para obtener más información sobre la interfaz de programación de aplicaciones DistributedObjectCache, consulte la documentación de la API.

Ámbito

Especifique el ámbito Célula para ver y configurar las instancias de antememoria que están disponibles para todos los servidores dentro de la célula. Especifique el ámbito NODO para ver y configurar las instancias de antememoria que están disponibles para todos los servidores dentro de un nodo determinado. Especifique el ámbito SERVIDOR para ver y configurar las instancias de antememoria que están disponibles sólo en un servidor determinado.

Nombre

Especifica el nombre de visualización necesario del recurso.

Nombre JNDI

Especifica el nombre de la interfaz JNDI (Java Naming and Directory Interface) para el recurso. Utilice este nombre cuando busque una referencia a esta instancia de antememoria. Como resultado se devuelve un objeto DistributedMap.

Descripción

Descripción opcional del recurso.

Categoría

Una serie de categoría opcional a utilizar cuando se clasifica o agrupa el recurso.

Tamaño de antememoria

Especifica un número entero positivo para el número máximo de entradas que contiene la antememoria. El tamaño de antememoria suele ser en miles. El valor predeterminado es 2000.

El valor mínimo es 100, sin un valor máximo establecido.

Prioridad por omisión

Especifica la prioridad predeterminada de las entradas de antememoria. Cuando la antememoria está llena, las entradas con prioridad más baja se eliminan de la antememoria antes que las entradas con prioridad más alta. Puede cambiar la prioridad de entradas de antememoria individuales mediante el archivo de políticas de antememoria. El valor predeterminado es uno. El rango oscila entre 1 y 255.

Habilitar la descarga de disco

Habilite la descarga de disco para especificar que cuando se llene la antememoria, se deberán eliminar entradas de la antememoria y guardarse en disco. La ubicación es una ubicación de directorio completa utilizada por la función de descarga de disco.

Ubicación de descarga de disco

Especifica el directorio que se utiliza para la descarga del disco. Este valor se ignora si la descarga de disco está inhabilitada.

Vaciado en disco

La opción Vaciado en disco especifica que cuando se detiene el servidor, el contenido de la antememoria debe trasladarse al disco. Este valor se ignora si la opción Habilitar la descarga de disco no está seleccionada.

Utilizar contexto de receptor

Establezca este valor en true para que los sucesos de invalidación se envíen a los receptores de invalidación registrados mediante el contexto J2EE (Java Platform, Enterprise Edition) del receptor. Si desea utilizar el contexto de la hebra de llamante para la devolución de llamada, establezca este valor en false.

Soporte de ID de dependencia

Especifica que el servicio de antememoria dinámica admite los ID de dependencia de entrada de antememoria. Si no necesita utilizar los ID de dependencia, inhabilite esta opción. Los ID de dependencia especifican identificadores de grupo de antememoria adicionales que asocian varias entradas de antememoria con el mismo identificador de grupo en la política de antememoria.

Es posible que esta opción no esté disponible para instancias de antememoria creadas con una versión anterior de WebSphere Application Server.

Habilitar la duplicación de antememoria

Utilice la duplicación de antememoria para habilitar la compartición de los ID de antememoria, entradas de antememoria e invalidaciones de antememoria con otros servidores del mismo dominio de duplicación.
