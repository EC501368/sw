Colección de cadenas de transporte del direccionador On Demand

Utilice esta página para ver o gestionar cadenas de transporte para el direccionador On Demand (ODR). Las cadenas de transporte permiten la comunicación mediante transportes o pilas de protocolo, que generalmente están basados en sockets.

Una cadena de transporte consta de uno o varios tipos de canales, cada uno de los cuales da soporte a un tipo diferente de protocolo de E/S, por ejemplo, TCP o HTTP. Los puertos de red se pueden compartir entre todos los canales de una cadena. La función Channel Framework distribuye automáticamente una petición que llega a ese puerto en el canal de protocolo de E/S correcto para que se procese.

La página Cadenas de transporte lista las cadenas de transporte definidas para el servidor de aplicaciones seleccionado. Las cadenas de transporte representan pilas de protocolos de red que operan dentro de este servidor de aplicaciones.

Para ver esta página de la consola administrativa, pulse Servidores > Server types > Direccionadores On Demand > nombre_odr > Puertos > Detalles. Pulse Ver transportes asociados para el puerto cuyas cadenas de transporte desea ver.

Hay que poseer privilegios administrativos de configurador o administrador para cambiar los valores de ODR.

Nombre

Especifica un identificador exclusivo para la cadena de transporte. El nombre del transporte debe ser exclusivo dentro de la configuración. Pulse el nombre de una cadena de transporte para modificar los valores de configuración.

Habilitado

Cuando se establece en true, la cadena de transporte se activa durante el arranque del servidor de aplicaciones.

Sistema principal

Especifica la dirección del protocolo Internet del host a la que se debe enlazar el transporte. Si el servidor de aplicaciones reside en una máquina local, el nombre de host podría ser localhost.

Puerto

Especifica el puerto al que se debe enlazar para el transporte. El número de puerto puede ser cualquier puerto que no se utilice actualmente en el sistema, puede ser localhost o el carácter comodín * (un asterisco). El número de puerto debe ser exclusivo para cada instancia de servidor de aplicaciones en una máquina determinada

Máximo de conexiones abiertas

Número máximo de conexiones que pueden estar abiertas al mismo tiempo en un canal de entrada.

Tiempo de espera de inactividad

Cantidad de tiempo (en segundos) que puede permanecer inactivo un socket antes de exceder el tiempo de espera de las peticiones de lectura o grabación. Después de leer los datos iniciales, este intervalo a veces lo alteran otros canales posteriores de la cadena de transporte.

Repertorio SSL

Nombre de la configuración SSL (Secure Sockets Layer) que este canal va a utilizar. Este canal utiliza los valores de SSL para negociar los protocolos de intercambio, descifrar datos y cifrar datos en conexiones creadas por un cliente.

Utilizar conexiones persistentes (mantener activas)

Al enviar un mensaje HTTP de salida, esta opción controla si por omisión se permitirán conexiones persistentes (Mantener activas) en lugar de una conexión que se cierra después del intercambio de petición / respuesta. Mientras esté habilitada, si el valor de la opción Máximo de peticiones persistentes es 1 o 0 se inhabilitarán las conexiones persistentes.

Máximo de peticiones persistentes

Número máximo de peticiones permitidas en una sola conexión HTTP.
-1
Indica peticiones ilimitadas por conexión.
0
Indica una sola petición por conexión; un número indica dicha la cantidad de peticiones por conexión.
El valor de conexiones persistentes (mantener activas) del usuario puede prevalecer sobre este valor.

Tiempo de espera de lectura

El intervalo de tiempo máximo en segundos que debe esperar mientras se lleva a cabo una lectura de datos en un socket.

Tiempo de espera de grabación

El intervalo de tiempo máximo en segundos que debe esperar mientras se lleva a cabo una grabación de datos en un socket.

Tiempo de espera persistente

El máximo tiempo desocupado en segundos en un socket entre las peticiones de cliente. Una vez transcurrido este tiempo de espera, la conexión se cierra.
