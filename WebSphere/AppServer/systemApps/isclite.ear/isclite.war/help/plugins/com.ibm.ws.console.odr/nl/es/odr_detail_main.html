Configuración del direccionador On Demand

Utilice esta página para modificar los valores de un direccionador On Demand (ODR) que ya existan.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidores > Direccionadores On Demand > nombre_odr. En la pestaña Configuración, puede editar los campos de valores de ODR.

Hay que poseer privilegios administrativos de configurador o administrador para cambiar los valores de ODR.

Para configurar el ODR de forma utilice sus opciones avanzadas de direccionamiento, como por ejemplo el direccionar peticiones a una célula en la que WebSphere Application Server no está en ejecución y habilitar la colocación en antememoria, pulse Propiedades del direccionador On Demand.

Nombre

Especifica un nombre lógico para el direccionador On Demand. Los nombres de los direccionadores On Demand deben ser exclusivos dentro de un nodo.

Se trata de un campo de sólo lectura.

Ejecutar en modalidad de desarrollo

Si se habilita esta opción puede reducirse el tiempo de arranque de un direccionador On Demand. Este tiempo puede incluir valores de JVM (Java Virtual Machine), como inhabilitar la verificación de código de bytes y reducir los costes de compilación JIT (Just-in-time). No habilite este valor en servidores de producción.

Seleccione recuadro de selección para reducir el tiempo de arranque de un ODR. Este tiempo puede incluir valores de JVM (Java Virtual Machine), como inhabilitar la verificación de código de bytes y reducir los costes de compilación JIT (Just-in-time). No habilite este valor en servidores de producción.

Habilite esta opción si desea utilizar los valores de JVM -Xverify y -Xquickstart durante el arranque. Después de seleccionar este recuadro de selección, guarde la configuración y reinicie el ODR para activar la modalidad de desarrollo.

Inicio paralelo

Seleccione este campo para iniciar el ODR en varias hebras. Esta opción puede reducir el tiempo de arranque.

Seleccione este recuadro de selección si desea que los componentes del ODR, los servicios y las aplicaciones se inicien en paralelo en lugar de secuencialmente.

El orden en el que se inicien las aplicaciones dependerá del peso asignado a cada una de ellas. Las aplicaciones que tenga el mismo peso se iniciarán en paralelo. Establezca el peso de una aplicación con la opción Peso inicial en la página de la consola administrativa Aplicación > Tipos de aplicación > Aplicaciones de empresa de WebSphere > nombre_aplicación.

Iniciar componentes según sea necesario

Seleccione esta propiedad si desea que los componentes del servidor se inicien tal y como son necesarios para una aplicación que se ejecuta en este servidor.

Si esta propiedad está seleccionada, los componentes del servidor se inician de forma dinámica según son necesarios. Cuando no se selecciona esta propiedad, todos los componentes de servidor se inician durante el proceso de arranque del servidor. Por consiguiente, si se selecciona esta opción puede mejorar el tiempo de arranque y reducir la cantidad de memoria que requiere el servidor, porque se inician menos componentes durante el proceso de arranque.

Iniciar componentes cuando son necesarios es más eficaz si todas las aplicaciones desplegadas en el servidor son del mismo tipo. Por ejemplo, esta opción resulta más eficiente si todas las aplicaciones son aplicaciones web que utilizan servlets y JavaServer Pages (JSP). Esta opción es menos eficaz si las aplicaciones utilizan servlets, JSP y EJB (Enterprise JavaBeans).

Nota: Para asegurar la compatibilidad con otros productos WebSphere, el valor predeterminado para esta opción está deseleccionado. Antes de seleccionar esta opción, compruebe que los otros productos WebSphere que se ejecutan conjuntamente con este producto admiten esta funcionalidad.
