Colección de direccionadores On Demand

Esta página lista los direccionadores On Demand (ODR) de la célula. Un ODR se encuentra en un nodo de Intelligent Management. Un ODR clasifica, otorga prioridades y direcciona tráfico SIP y HTTP en la empresa. Esta página sirve para crear, suprimir, iniciar y detener un ODR. Puede crear sus propias plantillas de servidor.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidores > Direccionadores On Demand.

Nombre

Especifica un nombre lógico para el direccionador On Demand. El nombre de los direccionadores On Demand deben ser exclusivos dentro de un nodo. Si hay varios nodos dentro de un clúster, los nombres de los direccionadores On Demand también deben ser exclusivos dentro del clúster. El nombre del direccionador On Demand se utiliza en tareas administrativas, por ejemplo para que los scripts puedan hacer referencia al servidor.

Nodo

Nombre del nodo en el que se encuentra el direccionador On Demand.

Versión

Indica el WebSphere® Application Server que está ejecutando.

Protocolo

Indica el protocolo o los protocolos que puede manejar el direccionador On Demand.

Este valor depende de los tipos de cadenas de transporte que se han configurado. Si las cadenas de transporte están configuradas de forma que terminan con SIPProxyInboundChannel, se muestra SIP. Si las cadenas de transporte están configuradas de forma que terminan con HTTPProxyInboundChannel, se muestra HTTP. Si ambos tipos de cadenas están configuradas, se muestra HTTP, SIP.

Estado

Indica si el direccionador On Demand está iniciado, detenido o si no está disponible. Si el estado es no disponible, quiere decir que el agente del nodo no está en ejecución en el nodo y que debe reiniciar el agente del nodo antes de poder iniciar el servidor.
