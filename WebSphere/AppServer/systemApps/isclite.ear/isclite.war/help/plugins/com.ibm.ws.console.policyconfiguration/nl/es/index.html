Configuración de políticas

Vaya a los temas de ayuda de esta sección.

  • Definir propiedades generales de la política de servicio
    Utilice esta página para crear políticas de servicio, que son los componentes básicos de una política operativa, de acuerdo con las directrices que se incluyen.
  • Definir propiedades objetivo de la política de servicio
    Utilice esta página para personalizar políticas de servicio con las directrices de esta página. Cuando se definen las propiedades de un objetivo, se puede determinar cuándo y durante cuánto tiempo estará en ejecución una política de servicio. Una política de servicio corresponde a un objetivo empresarial que se correlaciona con componentes de clase de trabajo y transacción. Una política de servicio clasifica en categorías y da prioridades a las solicitudes de trabajo. La política de servicio crea el objetivo, mientras que las clases de trabajo relacionan información especificada como URI (Uniform Resource Identifiers) a dicho objetivo. Las clases de trabajo de tipo IIOP (Internet Inter-ORB Protocol) utilizan Enterprise JavaBeans (EJB) y nombres de método EJB para correlacionarse con el objetivo. Las clases de trabajo de tipo JMS (Java Message Service) utilizan nombres de destino y buses para correlacionase con el objetivo.
  • Definir las pertenencias a la política de servicio
    Se puede definir la asociación de una política de servicio con una clase de transacción. Al combinar una política de servicio con las clases de transacción y de trabajo asociadas, el direccionador On Demand evalúa todo el trabajo de la cola en un momento determinado y toma decisiones acordes a los atributos de la política de servicio.
  • Definición de propiedades generales de clases de transacción
    Utilice esta página para crear nuevas clases de transacción, que son componentes de una política de servicio.
  • Colección de políticas de servicio
    Las políticas de servicio se crean y modifican de acuerdo a los objetivos empresariales que define el usuario. Las políticas de servicio son los componentes básicos para construir una política operativa. Una política operativa clasifica en categorías el trabajo y consta de una política de servido con afiliaciones a clases de transacción. El direccionador On Demand lee la política operativa para determinar el flujo de trabajo adecuado de acuerdo a los recursos de servidor disponibles.
  • Visión general de las políticas de servicio
    Existe la posibilidad de crear políticas de servicio que se adecúen a los objetivos empresariales que defina el usuario. Las políticas de servicio crean objetivos relacionados con el trabajo para clasificarlo en categorías y darle distintas prioridades.
  • Valores de política de servicio
    Utilice esta página para cambiar las políticas de servicio o las clases de transacción existentes, con las que se crea conjuntamente una política operativa.
  • Topología de políticas de servicio
    Existe la posibilidad de configurar el sistema para que muestre la topología de las políticas de servicio. Esta pantalla proporciona una correlación directa de las políticas de servicio específicas con clases de trabajo.
  • Colección de clases de transacción
    Existe la posibilidad de editar clases de transacción existentes, que son componentes de una política de servicio. Las clases de transacción aplican los objetivos empresariales de una política de servicio a URI de aplicación específicos.
  • Mover una clase de transacción
    Las clases de transacción se pueden mover de una política de servicio a otra. Las clases de transacción no pueden existir sin estar afiliadas a una política de servicio. Debido a esta restricción hay dos opciones si se desea finalizar relación entre una política de servicio y una clase de transacción: suprimir la clase de transacción o mover la clase de transacción a otra política de servicio. Se puede crear una política de servicio para mantener las clases de transacción que sea necesario guardar.
