Valores de política de servicio

Utilice esta página para cambiar las políticas de servicio o las clases de transacción existentes, con las que se crea conjuntamente una política operativa.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Políticas de servicio > política_servicio.

Hay que poseer privilegios administrativos de configurador o administrador para crear, modificar o eliminar clases de transacción y políticas de servicio.

Nombre

El nombre de la clase de política de servicio es un campo definido por el usuario que debe contener caracteres numéricos o alfanuméricos. Utilice nombres de políticas de servicio coherentes y con sentido, por ejemplo, PLATINUM, GOLD, SILVER y BRONZE.

Descripción

El campo de descripción complementa el campo de nombre y lo define el usuario. Este campo es opcional, pero en entornos con varios administradores del sistema, es mejor completarlo con información que sea de utilidad. Por ejemplo, un campo de descripción de una política PLATINUM podría incluir el texto, La política de servicio PLATINUM se reserva para el trabajo de prioridad más alta.

Tipo de objetivo

Este campo se puede seleccionar y puede ser uno de los cuatro valores siguientes:

  • Tiempo de respuesta medio: Indica el trabajo de alta prioridad. Este campo se utiliza con el valor del objetivo e importancia para limitar el tiempo que se mantiene el trabajo en la cola.
  • Tiempo de respuesta percentil: Este es el porcentaje de las peticiones que se manejan en un período de tiempo específico. Este campo se utiliza con el porcentaje de objetivo para definir el porcentaje de peticiones por segundos o milisegundos.
  • Discrecional: El valor por omisión para el tipo de objetivo. El valor Discrecional indica que el trabajo no tiene una prioridad alta y lo puede planificar el direccionador On Demand basándose en la carga de trabajo que existe en la cola.
  • Importancia Cuando el sistema está soportando mucha carga, el valor de importancia determina el trabajo de aplicación que puede tolerar mejor una disminución del rendimiento. Cuando mayor sea la importancia, menor tolerancia tendrá el trabajo asociado con dicha política de servicio a los bajones en el rendimiento. Para el ejemplo PLATINUM: seleccione Máxima como importancia. No establezca todas las políticas en el nivel de importancia más elevado salvo que desee que todas las políticas se degraden en la misma proporción cuando se sobrecargue el sistema.
Nota: Si la propiedad personalizada de la célula disableResponseTimeGoals=true, se mostrará un tipo de objetivo importance para los tipos de objetivo de política de servicio. Este tipo de objetivo se utiliza para establecer que sólo se utilizará el aspecto importance de la política de servicio.

Valor de objetivo

Establezca este valor en el tiempo máximo permitido para completar el trabajo. Por ejemplo, el trabajo PLATINUM no debe superar los dos segundos. El valor del campo objetivo no está disponible cuando el tipo de objetivo es discrecional.

Si se está utilizando un objetivo de tiempo de respuesta para tráfico SIP (Session Initiation Protocol), el valor del objetivo debe estar debajo de 500 ms.

Percentil de objetivo

Especifica el porcentaje de peticiones que se han de manejar en un período de tiempo específico. El valor del percentil objetivo puede ser un número entero de 1 a 99 inclusive. Este campo está disponible cuando el tipo de objetivo es el tiempo de respuesta percentil.

Importancia

Indica un nivel de importancia. El valor de importancia no está disponible cuando el tipo de objetivo es discrecional.

Supervisar las violaciones de política persistentes

Se necesitan distintos valores dependiendo del tipo de objetivo de política de servicio:

  • Vista de tiempo de respuesta medio
    • Valor del objetivo: Un entero que indica los milisegundos, segundos o minutos del valor del objetivo. Los valores aceptables van de 0 a 300000 milisegundos, de 0 a 300 segundos y de 0 a 2147483647 minutos.
    • Valor de periodo de tiempo: Un entero que indica los milisegundos, segundos o minutos después de los cuales se considerará incumplido el valor objetivo. Este valor puede ser de 0 a 1 día, inclusive.
  • Vista de tiempo de respuesta percentil
    • Percentil objetivo: Un entero que indica el porcentaje de solicitudes bajo el valor de objetivo que se han de supervisar. Este valor puede ser de 0 a 100 inclusive.
    • Valor de periodo de tiempo: Un entero que indica los milisegundos, segundos o minutos después de los cuales se considerará incumplido el valor objetivo. Este valor puede ser de 0 a 1 día, inclusive.

Pertenencia

Especifica una asociación entre las clases de transacción y la respectiva política de servicio. Cada política de transacción puede estar contenida sólo en una política de servicio. Se pueden realizar las siguientes acciones:
Nueva
Crea nuevas clases de transacciones que, a continuación, pueden convertirse en miembros de una política de servicio nueva o existente.
Eliminar
Si selecciona una política de transacción del campo de pertenencia y pulsa Eliminar, puede suprimir la asociación entre una política de transacción y una política de servicio. Esta acción suprime la política de transacción al mismo tiempo.
Modificar
Cambia una política de transacción existente. Para editar una política de transacción, en primer lugar debe seleccionar la política de transacción que desea editar y pulsar Modificar.
Mover
Mueve las clases de transacciones de una política de servicio a otra. Debido a que una clase de transacción no puede existir fuera de una asociación de política de servicio, la única opción que queda para mover es la de eliminar si no ya no desea que una clase de transacción y una política de servicio continúen afiliadas. Para mover una clase de transacción, seleccione la política que desee asociar a otra política de servicio y pulse Mover.
