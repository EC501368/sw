Definir propiedades generales de la política de servicio

Utilice esta página para personalizar políticas de servicio con las directrices de esta página. Cuando se definen las propiedades de un objetivo, se puede determinar cuándo y durante cuánto tiempo estará en ejecución una política de servicio. Una política de servicio corresponde a un objetivo empresarial que se correlaciona con componentes de clase de trabajo y transacción. Una política de servicio clasifica en categorías y da prioridades a las solicitudes de trabajo. La política de servicio crea el objetivo, mientras que las clases de trabajo relacionan información especificada como URI (Uniform Resource Identifiers) a dicho objetivo. Las clases de trabajo de tipo IIOP (Internet Inter-ORB Protocol) utilizan Enterprise JavaBeans (EJB) y nombres de método EJB para correlacionarse con el objetivo. Las clases de trabajo de tipo JMS (Java Message Service) utilizan nombres de destino y buses para correlacionase con el objetivo.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Políticas de servicio > Nueva.

Hay que poseer privilegios administrativos de configurador o administrador para crear, modificar o eliminar clases de transacción y políticas de servicio.

Propiedades de tipo de objetivo de tiempo de respuesta medio

Los objetivos de tiempo de respuesta medio son indicativos de trabajo con una prioridad más alta que el discrecional. Al objetivo de tiempo de respuesta medio se le asigna un objetivo de tiempo específico en los siguientes paneles.

Valor del objetivo
Este valor debe ser igual al tiempo máximo asignado para que finalice el trabajo según la política de servicio. Por ejemplo, el trabajo PLATINUM no debe superar los 1250 milisegundos. Un rango de objetivo válido debe ser mayor que 0 y menor o igual que 5 minutos. El máximo de cinco minutos se puede mostrar como 300 segundos o 300.000 milisegundos.
Importancia
Cuando el sistema está soportando mucha carga, el valor de importancia determina el trabajo de aplicación que puede tolerar mejor una disminución del rendimiento. Cuando mayor sea la importancia, menor tolerancia tendrá el trabajo asociado con dicha política de servicio a los bajones en el rendimiento. Para el ejemplo PLATINUM, seleccione Máxima como importancia. No establezca todas las políticas en el nivel de importancia más elevado salvo que desee que todas las políticas se degraden en la misma proporción cuando se sobrecargue el sistema.
Supervisar si se producen violaciones de política de servicio de forma persistente
Opcional. Requiere valores diferentes dependiendo del tipo de tiempo de respuesta.
Valor de objetivo delta
Especifica la diferencia de tiempo permitida entre el valor objetivo configurado y el tiempo de respuesta medio real de las solicitudes a las que se están dando servicio.
Valor de período de tiempo
Crea una tarea de tiempo de ejecución cuando se detecta que no se satisface la diferencia permitida que se ha configurado.

Propiedades de tipo de objetivo de tiempo de respuesta percentil

Los objetivos de tiempo de respuesta percentil son otra medida para el trabajo con una prioridad más alta que el discrecional. Los objetivos de tiempo de respuesta percentil se definen con un criterio específico en el siguiente panel. El tiempo de respuesta percentil objetivo es el porcentaje de peticiones cuyo tiempo de respuesta es T o inferior que debería ser P o mayor; un objetivo tiene valores concretos para T y P.

Percentil objetivo
El porcentaje de solicitudes que se han de manejar en un período de tiempo específico.
Valor del objetivo
Este valor debe ser igual al tiempo máximo asignado para que finalice el trabajo según la política de servicio. Por ejemplo, el trabajo PLATINUM no debe superar los 1250 milisegundos. Un rango de objetivo válido debe ser mayor que 0 y menor o igual que 5 minutos. El máximo de cinco minutos se puede mostrar como 300 segundos o 300.000 milisegundos.
Importancia
Cuando el sistema está soportando mucha carga, el valor de importancia determina el trabajo de aplicación que puede tolerar mejor una disminución del rendimiento. Cuando mayor sea la importancia, menor tolerancia tendrá el trabajo asociado con dicha política de servicio con los bajones en el rendimiento. Para el ejemplo PLATINUM, seleccione Máxima como importancia. No establezca todas las políticas en el nivel de importancia más elevado salvo que desee que todas las políticas se degraden en la misma proporción cuando se sobrecargue el sistema.
Supervisar si se producen violaciones de política de servicio de forma persistente
Opcional. Requiere valores diferentes dependiendo del tipo de tiempo de respuesta.
Porcentaje del objetivo delta
Especifica la diferencia porcentual permitida entre el percentil objetivo configurado y el porcentaje real de las solicitudes a las que se están dando servicio.
Valor de período de tiempo
Crea una tarea de tiempo de ejecución cuando se detecta que no se satisface la diferencia permitida que se ha configurado.

Propiedades de tipo de objetivo discrecional

Los objetivos discrecionales indican un trabajo sin un valor significativo. Como resultado, un trabajo de este tipo puede experimentar una degradación en el rendimiento cuando hay una restricción en los recursos.
