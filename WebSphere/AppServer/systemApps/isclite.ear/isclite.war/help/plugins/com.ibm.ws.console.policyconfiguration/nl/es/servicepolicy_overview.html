Visión general de las políticas de servicio

Existe la posibilidad de crear políticas de servicio que se adecúen a los objetivos empresariales que defina el usuario. Las políticas de servicio crean objetivos relacionados con el trabajo para clasificarlo en categorías y darle distintas prioridades.

Pulse Políticas operativas > Políticas operativas para ver esta página de la consola administrativa.

Hay que poseer privilegios administrativos de configurador o administrador para crear, modificar o eliminar clases de transacción y políticas de servicio.

Las políticas de servicio se utilizan para:
  • Clasificar en categorías el trabajo.
  • Asociar un objetivo de rendimiento a dichas clases de trabajo.
  • Definir la importancia relativa de conseguir un objetivo de rendimiento para dichas clases de trabajo.

Para clasificar el trabajo en categorías, se utiliza el concepto de clase de transacción. Las clases de transacción son componentes de una política de servicio. Una clase de transacción contiene URI (Universal Resource Identifiers) definidos por el usuario. Por ejemplo, todas las series URI /buy/*.html se podrían asociar a una clase de transacción denominada PayTransactionsClass. Las clases de transacción corresponden a la unidad de clasificación en categorías más pequeña del entorno de Intelligent Management. Los datos que se recopilan a este nivel pueden mostrar métricas de rendimiento con una granularidad correspondiente a una única clase de transacción.

El producto puede asociar objetivos de rendimiento a trabajos a nivel de clases de transacción mediante una política de servicio. Una política de servicio se define como una colección de clases de transacción que tiene asociado un objetivo de rendimiento. Por ejemplo, podría crear una política de servicio denominada Platinum y asociarle a PayTransactionClass como miembro. Defina objetivos de rendimiento asociados con clases de transacción en términos de tiempos de respuesta. El producto da soporte a varios objetivos de tiempos de respuesta, así como a objetivos de tiempos de respuesta percentiles.

Al definir una política de servicio, hay siete niveles distintos de importancia que se pueden utilizar. Si o se cumplen los objetivos de todas las políticas de servicio debido a una sobrecarga intensa y prolongada, se utiliza la importancia de criterios para decidir qué política de servicio tendrá más prioridad.
