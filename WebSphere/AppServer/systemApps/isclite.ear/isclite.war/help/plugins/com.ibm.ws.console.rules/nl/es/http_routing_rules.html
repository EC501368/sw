Normas HTTP de política de direccionamiento

Esta página permite asociar normas HTTP de política de direccionamiento con direccionadores On Demand (ODR) y clústeres ODR.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidor > Direccionadores On Demand > ODR > Propiedades del direccionador On Demand > Normas HTTP de política de direccionamiento.

Para ver esta página de la consola administrativa, pulse Servidores > Clústeres > Clústeres de direccionadores On Demand > Clúster ODR > Propiedades del direccionador On Demand > Normas HTTP de política de direccionamiento.

Hay que poseer privilegios administrativos de administrador para poder modificar normas con el generador de normas.

Añadir

Crea una regla nueva.

Suprimir

Suprime una norma.

Subir, Bajar

Da prioridad a las normas en el caso de varias normas. El orden de las normas es clave para determinar las coincidencias. Si se produce una coincidencia en la primera norma, las normas subsiguientes no se ejecutan.

Seleccionar acción

Las acciones hacen referencia a la solicitud HTTP de entrada.

Asigna la acción de direccionamiento por omisión a una de las opciones siguientes:
Permitir direccionamiento a
Permite direccionamiento a la solicitud HTTP de entrada.
Rechazar direccionamiento con código de error
Rechaza todas las solicitudes HTTP de entrada con el código de error especificado.
Permitir direccionamiento con afinidad a
Permite direccionamiento con afinidad a la solicitud HTTP de entrada.
Permitir direccionamiento a servidores en modalidad de mantenimiento
Direcciona la solicitud a los servidores sin ninguna interrupción.
Permitir direccionamiento con afinidad a servidores en modalidad de mantenimiento
Permite direccionamiento con afinidad sin ninguna interrupción.
Redirigir el direccionamiento a
Redirige la solicitud al URL especificado.
Direccionar al recurso local
Redirige la solicitud al sistema de archivos local del ODR desde el que se proporciona contenido estático.

Especificado por

Especifica clúster, servidor, módulo.

Seleccionar nombre de célula

Especifica la célula.

Política de direccionamiento con varios clústeres

El campo de política de direccionamiento con varios clústeres se visualiza al seleccionar cualquiera de las acciones de permitir. Además, el campo sólo se aplica si hay varios clústeres de destino. Si los clústeres destino de la política se calculan de forma dinámica, el número y el orden de los clústeres de destino se desconocen hasta el tiempo de ejecución.

Asigna la política de direccionamiento de varios clústeres con una de las siguientes opciones:
Sustitución por anomalía
Encuentra el primer clúster con un servidor disponible y equilibra la carga en dicho clúster. El orden de una lista generada de forma dinámica de clústeres no está definido.
Por turno circular con pesos
Equilibrio de carga por turno circular con pesos. En el caso de retransmisiones UDP, mantiene la afinidad.
Solicitud pendiente con el menor peso
Equilibrio de carga de solicitudes pendientes con el menor peso. En el caso de retransmisiones UDP, mantiene la afinidad.

URL de redireccionamiento

Este campo se muestra al seleccionar la acción de redireccionar. Con esta política de redireccionamiento, el ODR redirige la solicitud al URL especificado.

Código de rechazo

Esta campo se visualiza al seleccionar la acción de rechazar. Esta política de redireccionamiento hace que el ODR rechace la solicitud y devuelve el código de error HTTP especificado.

vía de acceso del recurso local

Este campo se muestra al seleccionar la acción de recurso local. Con esta política de redireccionamiento, la solicitud se redirige al sistema de archivos local del ODR desde el que se proporciona contenido estático.

Aplicar

Confirma los nuevos valores de la norma.
