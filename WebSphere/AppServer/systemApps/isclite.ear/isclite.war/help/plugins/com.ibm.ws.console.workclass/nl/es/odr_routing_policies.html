Políticas de direccionamiento del direccionador on demand

Utilice esta página para configurar políticas de direccionamiento para el direccionador on demand.

Para ver esta página de la consola administrativa, pulse Servidores > Tipos de servidor > Direccionadores On Demand > ODR > Propiedades del direccionador On Demand > Políticas de direccionamiento de clústeres de servidores genéricos.

Hay que poseer privilegios administrativos de administrador para poder modificar normas con el generador de subexpresiones.

Desde esta pestaña puede expandir Clases de trabajo HTTP para ver o gestionar las clases de trabajo para peticiones HTTP y lleve a cabo las acciones siguientes:

Clases de trabajo para peticiones HTTP

Expanda esta opción para ver o gestionar las clases de trabajo para peticiones HTTP (Hypertext Transfer Protocol).

Nueva

Crea una clase de trabajo nueva.

El asistente permite crear una nueva clase de trabajo y especificar propiedades generales.

Suprimir

Suprime una clase de trabajo.

Editar patrón HTTP

Se utiliza para añadir o suprimir un tipo de destino de la lista de miembros de clase de trabajo.

Añadir regla

Crea una norma nueva.

Consulte el Generador de subexpresiones para obtener más información.

Suprimir regla

Suprime una norma existente.

Subir, bajar

Da prioridad a las normas en el caso de varias normas. El orden de las normas es clave para determinar las coincidencias. Si se produce una coincidencia en la primera norma, las normas subsiguientes no se ejecutan.

Si la petición URI coincide con

Establece la condición de coincidencia URI (Uniform Resource Identifier) de una norma que ya exista. Escriba el nombre de URI en este campo.

Para el host virtual

Establece la condición de coincidencia URI de una norma que ya exista. Seleccione un nombre de host. Puede utilizar el carácter comodín asterisco (*) en el campo Si el URI coincide.

Seleccionar acción

Asigna la acción de direccionamiento por omisión a una de las opciones siguientes:
Permitir direccionamiento a
De la lista Seleccionar nombre de clúster de servidores genéricos, seleccione el clúster de servidores genéricos apropiados.
Rechazar direccionamiento con código de retorno
En el campo Entre el código de retorno, escriba el código de retorno HTTP.
Redirigir el direccionamiento a
En el campo Entre URI para redirigir a, escriba el URI.
Permitir direccionamiento con afinidad a
De la lista Seleccionar nombre de clúster de servidores genéricos, seleccione el clúster de servidores genéricos apropiados.

Aplicar

Confirma los nuevos valores de la norma.

Pulse Aplicar para confirmar el valor de regla.
