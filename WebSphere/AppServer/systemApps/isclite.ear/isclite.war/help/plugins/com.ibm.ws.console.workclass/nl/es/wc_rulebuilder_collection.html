Generador de normas

Utilice esta página para establecer la condición para una clase de transacción determinada para la aplicación y seleccione una clase de transacción con la que se correlacionan las peticiones. Para ver esta página, pulse Aplicaciones> Aplicaciones de empresa > aplicación > Políticas de direccionamiento. Expanda el tipo de petición y pulse Añadir norma > Generador de normas para una clase de trabajo existente.

En esta página pueden verse los atributos siguiente para el tipo de petición seleccionado:

Condición Pulse Añadir para establecer el operando, el operador y el valor de la norma para una condición de norma nueva o pulse una condición de norma existente para modificarla. Por ejemplo, si la norma utiliza un operando de ID de usuario establecido en Fred, la condición de norma se muestra como uid = 'Fred'.
Seleccionar acción Seleccione una de las opciones siguientes:
  • Permitir direccionamiento a: en la lista Seleccionar nombre de edición, seleccione el nombre de edición en que se permite el direccionamiento.
  • Permitir direccionamiento con afinidad a: en la lista Seleccionar nombre de edición, seleccione el nombre de edición en que se permite el direccionamiento.
  • Redirigir el direccionamiento a: En el campo Especificar URI para redireccionamiento, escriba el URI al que se redirigen las peticiones.
  • Rechazar direccionamiento con código de retorno: En el campo Especificar el código de retorno, proporcione el código de retorno del rechazo de direccionamiento.
Añadir Crea una norma.
Eliminar Suprime una norma seleccionada existente.
Borrar todo Borra todas las normas.
