Valores del controlador de ubicación de aplicaciones

Utilice esta página para definir los valores que controlan la ubicación de las aplicaciones. Se puede habilitar el controlador de ubicación autónomo, definir con qué frecuencia se darán cambios de ubicación y controlar el comportamiento del controlador de ubicación cuando opere en la modalidad supervisada.

Para ver esta página de la consola administrativa, pulse Políticas operativas > Gestores autónomos > Controlador de ubicación de aplicaciones.

Esta página describe los campos de la pestaña Configuración y de la pestaña Tiempo de ejecución. La pestaña Configuración sirve para configurar los valores que serán persistentes. La pestaña Tiempo de ejecución permite modificar parámetros que están activos mientras lo está el controlador de ubicación de aplicaciones. Si se inhabilita y a continuación se vuelve a habilitar el controlador de ubicación de aplicaciones, se utilizan los valores de configuración en lugar de los valores configurados en la pestaña Tiempo de ejecución.

Según sea el rol administrativo, existen privilegios específicos permitidos a la hora de configurar el controlador de ubicación autónomo. En la siguiente lista se muestran los roles administrativos y los privilegios para configurar el controlador de ubicación autónomo:

Supervisor
Ver la información.
Operador
Ver información en la pestaña de configuración. Cambiar los valores de la pestaña de tiempo de ejecución.
Configurador
Cambiar la configuración pero no los valores de tiempo de ejecución.
Administrador
Tiene todos los privilegios.

Habilitar seguridad. Cuando la seguridad está habilitada, algunos campos no se pueden editar sin la autorización de seguridad adecuada.

Habilitar

Habilita e inhabilita el controlador de ubicación de aplicaciones.

Si se inhabilita el controlador de ubicación de aplicaciones, inhabilita de forma efectiva todas las operaciones autónomas de los clústeres dinámicos. Una vez está inhabilitada la ubicación de aplicaciones, no se producirán cambios dinámicos en relación al tamaño y ubicación de aplicaciones en el clúster dinámico. Esta acción equivale a convertir todos los clústeres dinámicos en clústeres estáticos.

Tiempo de espera de aprobación

Controla cuánto tiempo está una tarea de tiempo ejecución en la cola esperando la aprobación o desaprobación antes de que dicha tarea caduque de forma automática.

Cuando un entorno operativo se configura para que se ejecute en una modalidad supervisada, el controlador de ubicación de aplicaciones crea tareas, pero espera la aprobación antes de realizar cualquier cambio. El controlador de ubicación de aplicaciones trata a las tareas de tiempo de ejecución que han excedido su tiempo de espera como tareas que el usuario ha rechazado. Los valores de tiempo aceptable están entre 1 y 60 minutos.

Para ver todas las tareas de tiempo de ejecución que ha creado la modalidad operativa supervisada, pulse Administración del sistema > Gestión de tareas > Tareas de tiempo de ejecución. Defina la notificación de correo electrónico si no desea supervisar la cola de tareas de tiempo de ejecución. Para crear un perfil de notificación de correo electrónico, pulse Administración del sistema > Gestión de tareas > Notificaciones.

Tiempo de espera de operación del servidor

Especifica el tiempo que espera el controlador de ubicación de aplicaciones para que se complete una operación de iniciar o detener antes que se considere errónea.

Los valores de tiempo aceptables están entre 1 y 60 minutos. Establezca este valor para predecir el peor caso en relación al tiempo que se necesita para iniciar o detener un servidor.

Tiempo mínimo entre cambios de ubicación

Especifica el tiempo que espera el controlador de ubicación de aplicaciones antes de iniciar un nuevo lote de cambios.

En algunos casos, el controlador de ubicación de aplicaciones espera a que se hayan completado sus últimos cambios o a que se exceda un tiempo de espera. Los valores aceptables están dentro del rango de 1 minuto a 24 horas. Al establecer este valor, considere la sobrecarga que se asocia con el inicio y la detención de los servidores. Se pueden tardar varios minutos en iniciar o detener servidores e introducir una carga adicional a los nodos.

La configuración de los controladores de ubicación para reajustar la ubicación de las aplicaciones causa muy a menudo una sobrecarga añadida en la que se pierde la ganancia de rendimiento que se habría ganado originalmente reajustando el tamaño de los clústeres dinámicos. Por ejemplo, si se tarda 1 minuto en iniciar un servidor y el valor para el tiempo mínimo entre cambios de ubicación es de 20 minutos, los cambios de ubicación afectan al rendimiento alrededor del 5%.

Hay que definir el parámetro para que sea al menos de 20 a 30 veces mayor que el tiempo necesario para iniciar un servidor. Un valor mayor de varias horas evita que los cambios de ubicación de aplicaciones se produzcan más de una vez cada día.

Guardar en el repositorio

Los campos en la pestaña Tiempo de ejecución son idénticos a los de la pestaña Configuración.

Pulse el recuadro de selección Guardar en el depósito para realizar los cambios en la pestaña Tiempo de ejecución y para guardar los valores en la configuración. De lo contrario, los cambios que se realizan en la pestaña Tiempo de ejecución no son persistentes, y únicamente existen mientras está activo el controlador de ubicación de aplicaciones.

Pulse Aplicar o Aceptar cuando haya realizado todos los cambios.
