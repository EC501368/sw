Grupos de nodos

Utilice esta página para editar grupos de nodos, que son agrupaciones lógicas de nodos definidas por el usuario con una propiedad compartida. Para ver esta página de la consola administrativa, pulse Administración del sistema > Grupos de nodos > seleccione un grupo de nodos existente para editarlo.

Esta página se divide en dos pestañas: configuración y topología local. La pestaña de configuración permite a un usuario ver los campos previamente configurados y, en algunos casos, editar esos campos. La pestaña de topología local permite al usuario ver las pertenencias del grupo de nodos.

Nota: cuando la seguridad está habilitada, algunos campos no pueden editarse sin la autorización de seguridad apropiada.

Campos de la pestaña Configuración:

NombreEl nombre del grupo de nodos es un campo definido por el usuario que debe contener caracteres numéricos o alfanuméricos. Este campo sólo es editable al crear nuevos nodos de grupo.
Número de miembrosEste campo está completamente automatizado y muestra el número de miembros del grupo de nodos. Aunque este campo no sea editable, se puede editar de forma indirecta añadiendo o eliminando miembros de nodos, utilizando el campo de pertenencia, cambiando así el número de miembros.
DescripciónEl campo de descripción es complementario al campo de nombre y lo define el usuario. Este campo es opcional, pero se recomienda, especialmente en entornos con varios administradores, que este campo incluya información útil.
PertenenciaEste campo se utiliza para añadir o eliminar nodos de un grupo de nodos. Existe una relación restrictiva que sólo permite a un nodo pertenecer a un único grupo de nodos, mientras que un grupo de nodos puede contener muchos nodos.

El campo de pertenencia se edita seleccionado los nodos adecuados; se pueden seleccionar varios nodos manteniendo pulsada la tecla CONTROL hasta que todas sus selecciones estén resaltadas, y pulsando Añadir o Eliminar para realizar los cambios de pertenencia deseados. Sin la autorización adecuada, la pertenencia no es editable si está habilitada la seguridad.

Una vez realizados todos los cambios, seleccione el botón Aplicar o Aceptar.

Pestaña de topología local:

La pestaña de topología local representa visualmente la jerarquía de pertenencia para el grupo de nodos. La estructura existe de forma que al nivel más alto está el grupo de nodos. Directamente bajo el grupo de nodos está la carpeta de nodos y la carpeta del clústeres dinámicos del grupo de nodos. Cada carpeta contiene los respectivos nodos o clústeres dinámicos.

La topología local no es un campo editable. Cualquier cambio de correlación de nodos miembros debe realizarse en la pestaña Configuración, en el campo Pertenencia.
