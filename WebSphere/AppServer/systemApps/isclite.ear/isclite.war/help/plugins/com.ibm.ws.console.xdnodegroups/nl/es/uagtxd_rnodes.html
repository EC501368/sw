Colección de nodos

Utilice esta página para gestionar los nodos del entorno WebSphere Application Server. Los nodos agrupan servidores gestionados.

Para ver esta página de la consola administrativa, pulse Administración del sistema > Nodos.

Nombre

Especifica un nombre para el nodo que es único en la celda.

Un nodo corresponde a un sistema físico con una dirección de host IP distinta. Generalmente, el nombre del nodo es el mismo que el nombre de host del sistema.

Ahora WebSphere Application Server admite el protocolo Internet versión 4 (IPv4) y el protocolo Internet versión 6 (IPv6), pero hay algunas limitaciones en cuanto al uso de ambos protocolos en la misma celda. Cuando se añade un nodo en una celda, el formato en el que se especifica el nombre se basa en la versión de IP que utilizará el nodo.

Versión

Especifica la versión de producto del nodo.

La versión de producto es la versión de un WebSphere Application Server para nodos gestionados. Para nodos no gestionados, en los que pueden definirse servidores Web, la versión se muestra como desconocida.

Protocolo de descubrimiento

Especifica el protocolo que utilizan los servidores para descubrir la presencia de otros servidores en el nodo.

Las opciones de protocolo posibles son:

UDP
Protocolo de datagrama de usuario (UDP)
TCP
Protocolo de control de transmisión (TCP)

Estado

Indica el estado del nodo.

El estado puede ser Iniciado, Detenido, No disponible, Desconocido, Inicio parcial, Parada parcial, No aplicable, Sincronizado o No sincronizado.

Modalidad de mantenimiento

Indica si el nodo se encuentra en modalidad de mantenimiento.

Puede llevar a cabo las acciones siguientes:
  • Establecer el mantenimiento con parada total para que un nodo quede fuera de línea, a la vez que se detienen todos los procesos del nodo para realizar tareas de mantenimiento, resolución de problemas o configuración de un nodo de copia de seguridad.
  • Establecer el mantenimiento para que un nodo quede fuera de línea, sin detener todos los procesos del nodo, para realizar tareas de mantenimiento, resolución de problemas o configuración de un nodo de copia de seguridad.
  • Anular la especificación del mantenimiento para que el nodo esté de nuevo en línea. Si el agente de nodo está detenido, reinícielo manualmente.
