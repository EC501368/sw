Resumen de informes

Puede utilizar diagramas personalizados para determinar si se están cumpliendo los objetivos empresariales y de rendimiento. Se puede realizar un seguimiento de estadísticas como la disponibilidad, el tiempo de respuesta, el tráfico y el rendimiento de distintos componentes de su entorno como, por ejemplo, las políticas de servicio, las aplicaciones, los servidores, los clústeres dinámicos y los nodos.

Para ver esta página de la consola administrativa, pulse Operaciones en tiempo de ejecución > Informes.

Preferencias de informe

Especifica las preferencias globales para todos los grupos de diagramas.

Grupo de diagramas por omisión
Especifica el grupo de diagramas que por omisión se muestra en esta página.
Tipo de diagrama predeterminado
Especifica el formato de la imagen del diagrama que se muestra. Se puede elegir entre SVG (Scalable Vector Graphics) o JPEG. Se necesita SVG para disponer la funcionalidad completa del diagrama. El navegador debe tener un plug-in SVG, o una implementación instalada, para ver diagramas en SVG.
Tipo de diagrama por omisión
Especifica el tipo de diagrama predeterminado al crear uno nuevo: línea, área, barra vertical, barra apilada vertical 3-D, barra horizontal, barra apilada horizontal 3-D. Cuando se cambia este valor, no se aplica a ningún otro diagrama que ya se haya creado. Pulse Preferencias en el diagrama para cambiar uno que ya exista.
Tamaño de diagrama por omisión
Especifica el tamaño de diagrama predeterminado cuando se crea un nuevo diagrama: pequeño, medio o grande. Cuando se cambia este valor, no se aplica a ningún otro diagrama que ya se haya creado. Pulse Preferencias en el diagrama para cambiar uno que ya exista.
Mostrar objetivos de la política
Al crear un nuevo diagrama, si se habilita, se muestra el valor de objetivo que se ha establecido para la métrica de datos, que se puede comparar con los valores actuales. Cuando se cambia este valor, no se aplica a ningún otro diagrama que ya se haya creado. Pulse Preferencias en el diagrama para cambiar uno que ya exista.
Mostrar tipos de conjuntos de datos
Si se habilita, se muestran los tipos de conjunto de datos para cada punto de datos. Cuando se cambia este valor, no se aplica a ningún otro diagrama que ya se haya creado. Pulse Preferencias en el diagrama para cambiar uno que ya exista.
Habilitar renovación automática
Si se habilita, los diagramas se renuevan de forma automática en cada intervalo de renovación de diagrama que se haya especificado.
Intervalo de renovación de diagramas
Especifica un número de segundos entre cada renovación de diagrama. Si se tiene un gran número de datos en los diagramas, considere establecer este valor con un intervalo mayor.

Botones de creación de diagrama

Para crear un nuevo diagrama dentro del grupo de diagramas, pulse Abrir una nueva pestaña de diagrama. Para abrir el diagrama seleccionado actualmente en una nueva ventana, pulse Ver diagrama en nueva ventana. Para devolver el diagrama desde la nueva ventana al grupo de diagramas, pulse Ver diagrama en grupo de diagramas.

Preferencias de diagrama

Pulse Preferencias para actualizar las preferencias del diagrama concreto. Estos cambios únicamente se aplican al diagrama seleccionado actualmente.

Tipo de diagrama
Cambia el aspecto del diagrama. Se pueden crear varios tipos distintos de diagramas de líneas y de barras:
  • Línea
  • Área
  • Barra vertical
  • Barra apilada vertical 3-D
  • Barra horizontal
  • Barra apilada horizontal 3-D
Tamaño de diagrama
Modifica el tamaño del diagrama. Se puede elegir pequeño, mediano o grande.
Mostrar tipos de conjuntos de datos
Especifica si se muestran los tipos de conjuntos de datos que se muestran en la columna Patrón de la columna de la tabla de datos de diagrama.
Mostrar objetivos de la política
Especifica si los datos se muestran junto con los objetivos que se han configurado.

Botones de datos de diagrama

Estos botones permiten cambiar los datos que se muestran en el diagrama y la tabla.

Añadir datos...
Añade conjuntos de datos al diagrama. Se pueden configurar varios conjuntos de datos para mostrarlos en el mismo diagrama. Después de pulsar Añadir datos..., cumplimente los siguientes campos:
Tipo de conjunto de datos
Especifica el tipo de objeto del que está realizando un seguimiento de métricas. Se puede seleccionar entre los siguientes tipos de conjuntos de datos:
  • Direccionador On Demand
  • Proxy
  • Grupo de nodos
  • Nodo
    Restricción: no es posible obtener la CPU, la memoria u otras estadísticas del nodo de gestor de despliegue. Si se añade un nodo de gestor de despliegue como un conjunto de datos, las estadísticas están en blanco.
  • Clúster dinámico
  • Clúster de servidores genéricos
  • Servidor de aplicaciones
  • Política de servicio
  • Clase de transacción
  • Aplicación
  • Módulo Java EE (Java Platform, Enterprise Edition=
  • Instancia de módulo Java EE
  • Nodo de base de datos
  • Servidor de base de datos
  • Usar ámbito actual como conjunto de datos
Este campo también se devuelve en la tabla del diagrama.
Conjunto de datos
Después de seleccionar un Tipo de conjunto de datos, se muestra una lista de los datos de los que puede realizar un seguimiento de métricas. Por ejemplo, si se elige Nodo como Tipo de conjunto de datos, se muestra una lista de nodos. Puede seleccionar un o más objetos en la lista. Este campo también se muestra en la tabla del diagrama.
Métrica de los datos
Esta opción se basa en un cálculo medio y es un conjunto por omisión de valores. Se pueden especificar las siguientes opciones para este valor:
  • Solicitudes simultáneas

    Esta métrica mide el número de solicitudes simultáneas para la selección del conjunto de datos definido en la selección de nombre de los datos.

  • Rendimiento medio

    La métrica de rendimiento medio se calcula en segundos basándose en el número total de solicitudes de flujo de trabajo enviadas al direccionador On Demand según se ha definido mediante el ámbito de datos, el nombre de los datos y el conjunto de datos.

  • Tiempo medio de respuesta (ms)

    La métrica de tiempo medio de respuesta hace referencia al tiempo que tarda en completarse una solicitud de trabajo. Esta métrica incluye la solicitud desde el principio, hasta que la solicitud se devuelve al cliente, y es el equivalente de la métrica del tiempo de servicio más el tiempo de espera en la métrica de cola.

  • Tiempo medio de espera en cola (ms)

    Esta métrica refleja el tiempo medio en milisegundos durante el que una solicitud de trabajo espera que se le dé servicio.

  • Tiempo medio de servicio (ms)

    Tiempo en milisegundos para prestar servicio a una solicitud del direccionador On Demand. Esta métrica indica el tiempo que tarda el direccionador On Demand en llevar el trabajo al nodo donde se completa.

  • Longitud media de cola

    El número promedio de solicitudes en cola por segundo para cualquier cola de trabajos en el entorno que se ha definido mediante el ámbito de datos, el conjunto de datos y el nombre de los datos.

  • Índice medio de eliminación

    El índice medio de eliminación se calcula en base al número de solicitudes de trabajo que no se procesan debido a que la cola está llena. Las solicitudes que encuentran una cola llena se devuelven al solicitante con un error.

  • Rendimiento medio relativo

    El rendimiento relativo del objetivo configurado en comparación con el valor del objetivo real.

  • Tiempo de respuesta percentil

    Porcentaje de solicitudes del objetivo de percentil de la política de servicio que cumplen el tiempo de respuesta especificado.

  • Solicitudes simultáneas

    El número de puestos necesarios para una carga de clúster dinámico dada procedente de una pasarela determinada.

  • Memoria utilizada

    La memoria utilizada por un proceso.

  • Utilización de CPU

    La CPU que utiliza un proceso.

  • Total de solicitudes

    El número total de solicitudes para los módulos Web desplegados para un proceso.

  • Tiempo de actividad

    El tiempo total en segundos que se ha estado ejecutando el proceso.

  • Número total de llamadas a métodos Enterprise JavaBeans (EJB)

    El número total de llamadas a los módulos EJB desplegados para un proceso.

  • Transacciones totales

    El número total de transacciones de una partición.

  • Memoria libre

    La memoria libre de un nodo.

  • Utilización

    Porcentaje de los recursos que se utilizan y cuántos recursos hay disponibles. La utilización mide la velocidad del nodo y la CPU de proceso.

  • Trabajos solicitados

    El número de trabajos que llegan al entorno de ejecución (aplicación de punto final) para su proceso.

  • Trabajos completados

    Número de trabajos que se ejecutan hasta su finalización en el entorno de ejecución.

  • Tiempo de ejecución

    Tiempo total, expresado en milisegundos, en que están en ejecución los trabajos.

  • Concurrencia máxima

    Nivel máximo de simultaneidad que se alcanza.

  • Trabajos en cola

    Número de trabajos que se encuentran en la cola del planificador.

  • Trabajos asignados

    Número de trabajos que ha asignado el planificador.

  • Trabajos no completados

    Número de trabajos que han dado un error en el entorno de ejecución.

  • Trabajos con error

    Número de errores de asignación que se han producido para los trabajos.

  • Tiempo de cola

    Tiempo en milisegundos durante el que un trabajo ha permanecido en cola.

  • Tiempo de asignación

    Tiempo total en milisegundos durante el que se han asignado trabajos.

  • Tiempo de error de asignación

    Tiempo en milisegundos durante el que se han asignado trabajos y se han producido errores de asignación.

  • Capacidad total de la CPU

    Un número de coma flotante de MHz del conjunto de instrucciones de referencia especificado para cada nodo.

  • Consumo de la CPU por los procesos de WebSphere

    El consumo de la CPU por parte de los procesos de WebSphere® en un nodo.

  • Consumo de CPU por otros procesos

    El consumo de la CPU por parte de otros procesos en un nodo.

  • Capacidad de la CPU disponible

    La capacidad de la CPU disponible de un nodo. Ésta es la capacidad total de la CPU menos el consumo por parte de los procesos de WebSphere y no de WebSphere.

  • Memoria total

    La memoria total de un proceso o nodo.

  • Coeficiente de trabajo

    Un factor de trabajo que caracteriza el promedio de trabajo de sistemas que se necesita para dar servicio a una solicitud de un tipo dado.

  • Demanda de la CPU

    La potencia de proceso necesaria para un destino de despliegue determinado.

  • Marca de límite de memoria alto

    El consumo de memoria previsto de una instancia de servidor que se va a iniciar.

  • Memoria residente

    La memoria residente de un proceso.

  • Coeficiente de potencia

    La cantidad media de potencia de cálculo consumida por parte de una solicitud activa de un flujo en una capa de proceso dada, promediada como (a) el tiempo transcurrido cuando la pasarela del gestor del flujo de solicitudes autónomo (ARFM) ha reenviado la solicitud al servidor de destino y (b) el tiempo que la respuesta tarda en volver del servidor de destino.

  • Capacidad autorizada

    El porcentaje de capacidad autorizada que esta siendo utilizado por una partición del servidor. Esta métrica sólo está disponible para microparticiones sin límite de potencia compartidas AIX.

  • Capacidad autorizada máxima

    Especifica el porcentaje máximo de capacidad autorizada que puede utilizar una partición de servidor. Esta métrica sólo está disponible para microparticiones sin límite de potencia compartidas AIX. El valor de esta métrica se calcula a partir de los ciclos desocupados que se encuentran en la agrupación de procesadores compartidos para una partición de servidor determinada.

  • Utilización de partición compartida
    Representa la utilización de procesador de la partición compartida. El valor se calcula con la fórmula siguiente: 
    (%capacidad autorizada / %máx capacidad autorizada) * 100
    Esta métrica sólo es válida para microparticiones sin limitación de potencia compartidas AIX.
  • Procesadores físicos consumidos

    Especifica el número de procesadores físicos que utiliza la partición de servidor compartida. Esta métrica sólo es válida para microparticiones sin limitación de potencia compartidas AIX.

  • Procesadores de agrupación disponibles

    Especifica el número de procesadores disponibles en la agrupación de procesadores compartida a la que pertenece esta partición de servidor. Esta métrica sólo es válida para microparticiones sin limitación de potencia compartidas AIX.

  • Índice de admisión de sesiones

    Especifica el índice de admisión de solicitudes sin afinidad de servidor, más el número de solicitudes de iniciación de sesiones.

  • Índice de no admisión de sesiones

    Especifica el índice de rechazo de solicitudes sin afinidad de servidor, más el número de solicitudes de iniciación de sesiones que han sido rechazadas.

Este campo también se muestra en la tabla del diagrama.
Mostrar datos únicamente de los direccionadores On Demand seleccionados
Al seleccionarlo, se podrán elegir varios direccionadores On Demand de la lista. El diagrama muestra datos sólo de los direccionadores On Demand seleccionados.
Cambiar ámbito...
Filtra la lista de objetos de conjuntos de datos disponibles para el diagrama. Por ejemplo, seleccionando el ámbito de clúster dinámico y un clúster dinámico existente, puede ver objetos específicos únicamente para el clúster dinámico seleccionado. Si se cambia el ámbito, se eliminarán todos los conjuntos de datos actuales y se restablecerá el ámbito del diagrama. Se puede elegir un determinado tipo de objeto, por ejemplo, un clúster dinámico y, a continuación, elegir una instancia de objeto específica. Si se elige un clúster dinámico, se muestra una lista de clústeres dinámicos.

Elija la instancia de objeto y pulse Aceptar, salvo que seleccione el ámbito de Component stability. El ámbito por omisión es Component stability y no se realiza ningún filtrado en la lista de objetos de conjuntos de datos disponibles para el diagrama.

Eliminar
Elimina los datos seleccionados del diagrama. Seleccione los datos que desee eliminar de la tabla y pulse Eliminar.
Tabla de vista
Muestra una tabla de los datos en el diagrama.

Tabla de diagrama

La tabla de diagrama muestra una lista de datos que se incluyen en el diagrama.

Patrón
Especifica la forma que se visualiza en el diagrama, si se está utilizando un diagrama de líneas. No se puede cambiar el patrón.
Tipo de conjunto de datos
Especifica el tipo del conjunto de datos con los que está realizando un seguimiento de las métricas.
Conjunto de datos
Indica el objeto que está proporcionando los datos en tiempo de ejecución. Pulse Añadir datos... para añadir nuevos objetos para realizar su seguimiento.
Métrica de los datos
Visualiza las métricas seleccionadas para realizar el seguimiento para este conjunto de datos.
Filtro de datos
Con la ayuda del campo de Filtro de datos se pueden seleccionar proxies existentes o direccionadores On Demand y filtrar la correlación consecuentemente. Esta selección resulta útil para la resolución de problemas del entorno.
Escala
Indica el tamaño de los datos representados.

Opciones de grupo de diagramas

Para guardar el grupo de diagramas, entre un nombre y pulse Guardar.

Para eliminar el actual grupo de diagramas, pulse Eliminar grupo de diagramas.
