Resumen de aplicaciones

Esta página de la consola administrativa muestra un resumen operativo de todas las aplicaciones iniciadas de su entorno, incluido el estado, la estabilidad y las asociaciones de política de servicio activas.

Para ver esta página de la consola administrativa, pulse Operaciones en tiempo de ejecución > Aplicaciones.

Diagrama de resumen de aplicaciones

El diagrama muestra el tiempo de respuesta medio de las aplicaciones que se están ejecutando.

Por omisión, el número máximo de conjuntos de datos que se visualiza es 5.

Nota: si la aplicación no tiene carga en la célula de WebSphere ® Deployment, el diagrama de resumen estará vacío. Se trata del comportamiento esperado. Para ver datos en el diagrama hay que comenzar a ejecutar carga con las aplicaciones.

Tabla de resumen de aplicaciones

La tabla de resumen de la aplicación muestra información sobre las aplicaciones iniciadas de su entorno.

Pulse Preferencias e indique un Número máximo de filas para limitar el número de filas que se muestran en la tabla. Pulse Aplicar. Utilice Restablecer para utilizar de nuevo el valor por omisión. Utilice el icono de filtro compuesto para filtrar la información que se muestra en la tabla.

Nombre

Especifica el nombre de la aplicación.

Pulse sobre el nombre de la aplicación para ver detalles de la configuración e información sobre operaciones y creación de informes adicionales.

Célula

Especifica el nombre de la célula.

Edición

Especifica la edición de la aplicación.

Podría haber varias ediciones en ejecución de una única aplicación. Pulse Aplicaciones > Centro de control de ediciones para gestionar las distintas ediciones de una aplicación.

Tipo

Especifica el tipo de la aplicación. Entre los tipos de aplicación se incluyen: Java EE (Java Platform, Enterprise Edition), PHP, WebSphere Application Server Community Edition y aplicaciones Web no gestionadas.

Estado

Indica si la aplicación desplegada en el servidor de middleware está iniciada, detenida o no está disponible.

Tabla 1. Estado de la aplicación de middleware
Icono Estado Descripción
Iniciada parcialmente La aplicación está en el proceso de cambiar de un estado de Detenida a un estado de Iniciada. Se está comenzando a ejecutar la aplicación pero todavía no lo está haciendo en su totalidad. La aplicación no se puede iniciar en su totalidad porque un servidor correlacionado con uno o más módulos de aplicación está detenido.
Iniciado La aplicación se ha iniciado.
Detenida parcialmente La aplicación está en el proceso de cambiar de un estado de Iniciada a un estado de Detenida. La aplicación todavía no se ha detenido.
Detenido La aplicación no está en ejecución.
Desconocida No se puede determinar el estado.

Una aplicación con un estado de no disponible puede estar en ejecución, pero tiene un estado de no disponible porque el servidor que ejecuta la consola administrativa no se puede comunicar con el servidor en el que la aplicación está en ejecución.

Estabilidad

Especifica el estado operativo de la aplicación.

Tabla 2. Estabilidad de la aplicación de middleware
Icono Estado Descripción
Estable La aplicación está operativa y sin problemas.
Cuestionable La aplicación tiene algunos problemas, pero sigue operativa.
Inestable La aplicación tiene graves problemas.
Desconocida No se puede informar sobre la estabilidad de la aplicación.

Asociaciones de políticas de servicio activas

Especifica todas las políticas de servicio, a lo largo de la vida de los direccionadores On Demand (ODR), con las que se ha clasificado el trabajo de la aplicación.
